## 📌 좋아요 3천개 받은 역대급 MCP는 무엇이며, 어떤 기능들을 제공하는가?
 이번 영상에서 소개하는 역대급 mcp는 <mark>Task </mark>mcp<mark>(</mark>슈림프 테스크 매니저<mark>), </mark>에디트 파일라인즈<mark> </mark>mcp<mark>, 깃 </mark>mcp 세 가지이며, 이들은 작업 관리, 문서 편집, 버전 관리를 통해 생산성을 크게 향상시킵니다. 


## 💡 각 MCP의 주요 기능은 무엇인가?

- **슈림프 테스크 매니저**: 작업 계획, 분석, 평가, 하위 작업 분할, 우선순위 지정, 검증 등 스마트한 직원처럼 작업을 관리합니다.
- **에디트 파일라인즈**: 정규 표현식을 사용하여 파일의 특정 부분을 정확하게 찾아 편집하고, 드라이런 기능으로 미리 결과를 확인할 수 있습니다.
- **깃 ****MCP**: 모든 작업 내용을 저장하여 문제가 생겼을 때 이전 상태로 쉽게 돌아갈 수 있게 하며, 브랜치 기능을 통해 효율적인 버전 관리를 돕습니다.

생산성을 **극적으로 끌어올리는** 세 가지 필수 mcp(Mega Custom Prompt) 조합을 소개합니다. 특히 api 키 없이 **무료로 사용 가능한** '슈림프 테스크 매니저'를 중심으로, 복잡한 프로젝트 관리, 문서 편집, 그리고 안전한 코드 관리를 위한 Git mcp까지 완벽하게 설정하는 방법을 배울 수 있습니다. 이 콘텐츠를 통해 마치 **똑똑한 직원처럼 스스로 계획하고 실행하며 검증**까지 수행하는 AI 작업 환경을 구축하고, 개발자는 물론 문서 작업이 많은 분들도 체계적인 작업 관리와 효율적인 문제 해결 능력을 갖출 수 있습니다.
## 1. 생산성을 극대화하는 세 가지 필수 MCP 소개 및 배경 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/pNkyZE780xE/0.jpg)
1. **MCP****(Mega Custom Prompt)의 중요성 강조** 
  1. 소개되는 세 가지 MCP를 설치하면 체감상 생산성이 상당히 올라간다고 언급된다. 
  2. 이전 MCP 영상들이 <mark>좋아요 3,000개</mark> 이상을 받았으며, 이번 MCP는 이전보다 더 업그레이드되었다고 밝힌다. 

2. **설치 지원 도구 소개** 
  1. 설정하기 쉽도록 **설치 도움이 GPT**를 만들었다. 
  2. 이 도우미를 통해 필요한 MCP를 출연내고 쉽게 설치할 수 있으며, MCP 관련 막히는 부분도 잘 알려줄 것이라고 설명한다. 

3. **'****슈림프 테스크 매니저****' ****MCP**** 선택 배경** 
  1. 유튜브에서 유명한 '테스크 마스터 MCP'는 성능이 뛰어나 '인생 MCP'로 불리지만, 설치 시 **API 키****가 필요**하고 사용할 때마다 비용이 발생하는 단점이 있다. 
  2. 이에 따라 API 키가 필요 없는 **무료 ****MCP**인 **'****슈림프 테스크 매니저****'**를 찾았으며, 이는 테스크 마스터 MCP와 비슷하게 작동한다. 
  3. 슈림프 테스크 매니저는 설치가 쉽지 않았으나 모든 문제를 해결했으며, 그 방법을 알려줄 것이라고 약속한다. 
  4. 이 MCP는 개발자가 아니더라도 유용하며, 특히 **문서 작업을 많이 하는 분들**에게 완전 추천하고 개발자에게는 필수품으로 느껴질 것이라고 강조한다. 


## 2. 슈림프 테스크 매니저 MCP의 기능 및 작동 방식 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/pNkyZE780xE/68.jpg)
1. **핵심 기능: ****작업 관리**** 및 분석** 
  1. 슈림프 테스크 매니저는 작업을 관리해 주며, 작업 목록을 요청하면 완료된 일과 진행 중인 일을 보여주고, 각 작업을 클릭하면 자세한 내용을 알 수 있다. 
  2. **프로젝트 지침**을 통해 작업을 계획하면 MCP가 다음 기능을 수행한다. 
  1. **세부 작업 생성**: MCP가 세부 작업을 만들고 각 작업이 완료되는 조건까지 설정한다. 
  2. **작업 분석**: 문서를 검토하고 위험 요소를 미리 파악한다. 
  3. **추론 기능**: 복잡한 문제를 논리적으로 단계별로 생각하며, 가설을 세우고 검증하는 과정을 거친다. 
  4. **반성적 평가**: 계획 진행 상황을 평가하고 개선할 부분을 알려준다. 
  5. **작업 분할 및 우선순위**: 작업이 너무 크면 작은 하위 작업으로 나누고, 우선순위를 정해 무엇을 먼저 할지 정리해 준다. 
  6. **자율적 판단 및 검증**: 사용자가 무언가를 시키면 똑똑한 직원처럼 필요한 것을 스스로 고민하고, 상황을 보고 위험 요소와 작업 순서를 체계적으로 판단하며, 작업이 끝나면 제대로 됐는지 검증하는 단계까지 수행한다. 

2. **클로드 내 역할 분담을 통한 효율성 증대** 
  1. 클로드에게 두 가지 역할을 부여하여 효율성을 높였다. 
  1. **테스크 플래너 역할**: 계획을 세울 때 사용된다. 
  2. **테스크 실행자 역할**: 실제 작업을 수행할 때 사용된다. 
  2. 쉬림프로 계획을 세울 때는 테스크 플래너가, 실행을 할 때는 테스크 실행자가 일을 하게 되어 아주 효율적이다. 

3. **작업 처리 과정 및 장점** 
  1. MCP에 작업을 등록해 달라고 부탁하면, MCP는 작업 계획 전에 관련 소스를 살펴본다. 
  2. 그 후 작업을 계획, 분석, 평가하고 작은 작업으로 나눈 뒤 본격적으로 작업을 시작한다. 
  3. 이 테스크 MCP는 할 일을 알아서 처리해 주어 편하고 결과도 좋으며, 기능도 만족스럽다. 
  4. **주요 장점**:
  1. 프로젝트 플랜 같은 파일이 필요 없이 모든 것을 알아서 처리해 준다. 
  2. 이전 작업들을 기억하며, 길이 제한에 걸려 새 창을 열어도 작업 목록을 그대로 볼 수 있다. 

4. **사용 시 주의할 점** 
  1. 새 창에서 시작할 때 슈림프 MCP가 초기화를 물어볼 때가 있다. 
  2. 초기화는 **처음에만** 해야 하며, 다시 초기화하면 이전 작업들이 사라지므로 조심해야 한다. 


## 3. 슈림프 테스크 매니저 사용법 및 연동 방법 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/pNkyZE780xE/207.jpg)
1. **간단한 사용 절차** 
  1. 프로젝트를 생성하고 지침을 입력한다. 
  2. 슈림프 MCP 초기화를 부탁한다. 
  3. 할 일을 등록해 달라고 하면 작업이 등록되며, 여러 작업을 등록할 수도 있다. 
  4. 등록된 작업을 진행하라고 하면 MCP가 알아서 일을 시작하고, 작업이 끝나면 스스로 검증 후 완료 처리한다. 
  5. 이 과정을 통해 작업 관리가 더 체계적이게 된다. 

2. **클로드와 ****커서****(Cursor) 간의 작업 연동** 
  1. 클로드에서 작업하다가 커서로 이어 작업할 수 있다. 
  2. 커서 프로젝트 파일이 있는 폴더를 열고 쉬림프 작업 목록을 살펴보라고 하면 클로드의 작업이 커서에서 이어진다. 
  3. 반대로 커서의 작업을 다시 클로드로 옮겨 이어갈 수도 있다. 
  4. 다만, 이렇게 하려면 클로드와 커서의 설정이 동일해야 한다. 

3. **기존 프로젝트 적용 및 설치 안내** 
  1. 이미 진행 중인 프로젝트에도 슈림프 MCP 적용이 가능하다. 
  2. 쉬림프로 초기화를 요청하면 기존 소스를 분석하여 세팅해 주며, 이후 쉬림프 MCP를 통해 작업을 추가하면 된다. 
  3. 세팅 방법은 영상 마지막 부분에서 알려줄 것이라고 예고한다. 


## 4. 에디트 파일라인즈 (Edit File Lines) MCP 소개 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/pNkyZE780xE/264.jpg)
1. **새로운 문서 편집 ****MCP****의 필요성** 
  1. 이전에 소개한 텍스트 에디터 MCP는 특정 텍스트를 교체할 때 같은 표현이 문서 내에 여러 개 있으면 오류가 나는 불편한 점이 있었다. 
  2. 이에 따라 새로운 문서 MCP인 **'****에디트 파일라인즈****'**를 소개한다. 

2. **주요 장점: ****정규 표현식**** 및 ****드라이런**** 기능** 
  1. 이 MCP의 좋은 점은 **정규 표현식**을 사용할 수 있다는 것이다. 
  2. 정규 표현식을 통해 파일에서 특정 부분만 정확하게 찾아낼 수 있으며, 예를 들어 이닛(init) 함수만 수정하고 싶으면 MCP가 그 부분만 정확히 찾을 수 있다. 
  3. 필요한 부분만 확인하고 수정하니 경제적이다. 
  4. **드라이런****(Dry Run)** 기능이 있어 편집 결과를 미리 보고 문제 없을 때 최종 처리할 수 있으며, 문서 편집기들은 자주 오류가 나기 때문에 이 기능이 꼭 필요하다. 

3. **사용 시 문제점 및 해결책** 
  1. 이 MCP에도 문제가 있는데, 한 번 편집하면 라인이 변경되는데 MCP가 변경된 라인을 인식하지 못할 때가 있다. 
  2. **해결책**: 편집하기 전에 항상 관련 부분을 조회하도록 해야 하며, 이렇게 하면 라인이 변경돼도 문제없이 작업할 수 있다. 
  3. 이 내용은 프로젝트 지침에 넣어두면 좋지만, 지침에 넣어도 종종 잊어버리니 **첫 프롬프트에서 알려 주는 것**이 필요하며, 이후로는 잘 진행될 것이다. 


## 5. 깃 (Git) MCP 소개: 코드 관리 및 시간 여행 기능 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/pNkyZE780xE/334.jpg)
1. **깃 ****MCP****의 필요성** 
  1. 깃(Git)은 개발자들에게 익숙한 도구이다. 
  2. AI로 코딩할 때 기능이 꼬여 이전으로 돌아가지 못하는 경우가 발생할 수 있으며, 이런 문제를 해결하기 위해 깃 MCP를 사용한다. 

2. **커밋(Commit)****을 통한 작업 저장 및 복구** 
  1. 깃은 모든 작업 내용을 저장하며, 문제가 생기면 잘 작동하던 상태로 쉽게 돌아갈 수 있다. 
  2. 이는 게임에서 중간중간 세이브하여 보스와 싸우다 죽으면 이전 세이브 상태로 로드하는 것과 유사하다. 
  3. 깃도 과거 내역 중 어느 하나로 돌아갈 수 있는데, 이는 깃이 **커밋(Commit)**이라는 단계로 작업을 저장하기 때문이다. 
  4. 파일을 수정한 후 커밋하면 그 내용이 저장되며, 이후 문제가 생기면 저장된 커밋으로 돌아갈 수 있다. 
  5. VS 코드와 함께 사용하면 지난 수정 내역을 볼 수 있어, 깃 덕분에 코드 변경 사항을 눈으로 확인하며 관리할 수 있다. 

3. **브랜치(Branch)**** 개념을 통한 효율적인 개발** 
  1. 깃에는 **브랜치**라는 개념이 있다. 
  2. 필자는 마스터 브랜치와 테스트 브랜치를 사용하고 있다. 
  3. **마스터 브랜치**는 실전 모드이고, **테스트 브랜치**는 연습 모드와 같다. 
  4. 테스트 브랜치에서 자유롭게 코딩하고 잘 작동하면 마스터 브랜치에 합치는데, 이것을 **머지(Merge)**라고 한다. 
  5. 반대로 테스트 중 문제가 생기면 마스터 브랜치 내용을 다시 가져올 수도 있다. 
  6. 브랜치를 전환하면 폴더의 파일들이 해당 브랜치 상태로 일괄 변경된다. 
  7. 깃으로 과거의 특정 시점으로 돌아가도 그 시기의 내용으로 일괄 변경되며, 이는 마치 시간 여행을 하는 느낌이다. 
  8. 깃으로 과거로 돌아갈 수 있지만, 브랜치를 나누면 더 효율적이며, 익숙해지면 마스터, 개발, 기능 개발 단위의 기능 브랜치, 실험 기능 브랜치 등 여러 브랜치로 관리할 수 있다. 


## 6. MCP 조합의 효과 및 설치 방법 안내 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/pNkyZE780xE/453.jpg)
1. **MCP**** 조합의 효과** 
  1. 이 MCP들은 개발자들에게 특히 유용하지만, 문서 작업을 할 때도 모두에게 도움이 된다. 
  2. 프로젝트를 시작할 때 이렇게 설정하면 계속 버튼만 눌러도 작업이 쉽게 끝나며, 확실히 편해진 느낌이다. 
  3. 문서 리서치는 한 번에 끝날 수도 있으며, 코딩은 버그 수정이 더 필요할 수 있지만 작업이 확실히 편해졌다. 

2. **슈림프 테스크 매니저**** 설치 방법** 
  1. MCP 세 개의 설치 방법은 댓글에 적어 놓을 것이며, 슈림프 테스크 마스터는 따로 설명한다. 
  2. **설치 단계**:
  1. 명령 프롬프트를 열고 아래 명령어를 순서대로 입력한다. 
  2. 설치가 끝나면 인덱스 JS 파일을 편집기로 연다. 
  3. 파일에서 콘솔 로그라고 적힌 부분을 모두 주석 처리하거나 삭제한다. 
  4. 작업 서버로 XAMP를 사용했으며, XAMP 설치 방법은 이전 영상을 참고해야 한다. 
  5. 쉬림프 설치 폴더 안에 있는 프롬프트 폴더를 복사한다. 
  6. 홈페이지 루트 폴더인 마이트에 붙여 넣는다. 
  7. 프롬프트 폴더의 이름을 **쉬림프**로 바꾼다. 
  8. 클로드 데스크탑의 설정 파일에 다음 내용을 추가하고, 댓글로 알려줄 프로젝트 지침도 빠뜨리지 않아야 한다. 

3. **MCP**** 설치 도움이 GPT 활용** 
  1. 설치가 어려우면 **MCP**** 설치 도움**을 이용하면 되며, 이는 GPTS이다. 
  2. 이곳에 들어가서 궁금한 점을 물어보거나, 필요한 작업 내용을 말하고 어떤 MCP가 필요한지 물어볼 수 있다. 
  3. 설치 과정에서 생긴 어려운 점을 질문해도 좋으며, 특정 MCP만 설치하고 싶다면 그렇게 요청하거나 클로드 데스크탑과 같은 작업 환경을 알려주면 그에 맞게 설치법을 알려줄 것이다. 

4. **마무리** 
  1. 앞서 설명한 내용은 댓글로도 정리해 줄 것이며, 새롭고 심박한 영상이 계속될 것이라고 예고하며 마무리한다.
