## 1. **기존 AI 코딩의 한계 극복 및 새로운 AI 코딩 방법 소개** 


### 1.1. **새로운 AI 코딩 방법의 필요성 제기** 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/GRtrfdSeR20/0.jpg)
1. **지브리 스타일 그림 생성 GPT-4 연구 사례 소개**: AI가 웹에서 자료를 검색하고 스스로 연구하여 웹페이지 결과물을 만들어내는 방식이다. 
  1. 생성된 자료는 양이 방대하고 퀄리티가 높으며 신뢰도까지 우수하다. 
  2. 일반적인 클로드 MCP만으로는 이러한 방대하고 깊이 있는 결과물을 얻기 어렵다는 점에서 놀라움을 주었다. 
2. **기존 AI 코딩의 문제점**: 
  1. 초기에는 좋지만, 유지보수가 매우 어렵다. 
  2. 기능이 복잡해질수록 버그가 발생하고, 직접 작성하지 않은 코드의 경우 문제 발생 시 수정이 어렵다. 
3. **새로운 AI 코딩 방법의 목표**: 기존 AI 코딩의 치명적인 단점을 완벽히 해결하여 AI와 인간의 진정한 협업을 구현하는 것이다. 


### 1.2. **새로운 AI 코딩 방법의 실제 적용 및 과정** 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/GRtrfdSeR20/39.jpg)
1. **AI의 자동화된 코딩 과정**: 
  1. AI가 웹사이트 접속 및 로그인, 테스트, 버그 수정 등을 자동으로 진행한다. 
  2. 유튜브 쇼츠 자동 생성 프로그램 개발 시 이 기법을 활용하여 대용량 소스를 개발했다. 
  3. MCP를 이용하여 브라우저 자동 제어, 콘솔 로그 체크, 스크린샷 촬영, 서버 에러 확인 등을 AI가 스스로 수행하도록 만들었다. 
  4. AI가 직접 데이터베이스를 분석하고 관리하는 것도 가능하다. 
2. **버그 발생 시 AI 활용 전략**: 
  1. 쇼츠 자동 생성 실행 중 에러가 발생하면, AI에게 에러 상황을 설명하고 원인 조사를 지시한다. 
  2. **중요한 지침**: AI에게 "일단 조사하고 기다려라"라고 명령하여 성급한 판단을 제어한다. 
  3. AI는 스스로 원인을 분석하여 보고하며, 이중 세션 관리 문제를 파악한다. 
  4. AI에게 "세션 관리 소스를 꼼꼼하게 살펴보고 더 깊이 고민하라"고 지시한 후 다시 "기다려라"고 명령한다. 
  5. 이러한 과정을 통해 버그 원인이 더 명확해지고, 클로드가 문제를 더 잘 이해하게 된다. 
  6. **AI 제어 노하우**: 
  1. AI에게 전적으로 맡길 경우 대규모 소스에서 혼란에 빠지거나 작업을 무한 반복할 수 있으므로, 중간중간 "기다려라"고 명령하는 것이 중요하다. 
  2. AI가 제시한 내용을 바탕으로 더 깊이 있는 질문을 던지면 클로드가 더욱 정확하게 문제를 파악하게 된다. 
  3. 예시로 AI는 PHP 세션과 자바스크립트 API의 관계 및 문제점을 구체적으로 알려준다. 
  4. 질문을 던질수록 AI는 점점 더 정확한 분석을 수행한다. 
  7. AI의 분석을 통해 버그 원인을 파악한 후, 개발자가 해결 방법을 제시한다. 
  8. 마지막으로 AI에게 "작업이 끝날 때까지 멈추지 말고 계속 진행하고 완료되면 기다려라"고 지시한다. 
3. **작업 관리 및 재개 시스템**: 
  1. AI는 개발자의 지시에 따라 수많은 작업을 일괄적으로 진행한다. 
  2. 작업 완료 후 **프로젝트 플랜 파일**을 업데이트하는데, 이 파일은 프로젝트의 현재 상황, 진행된 작업, 다음 작업 등을 상세히 기록한다. 
  3. MCP를 통해 AI가 생성한 모든 파일은 컴퓨터에 자동 저장된다. 
  4. 작업 중 AI가 멈추거나 에러가 발생해도, 다음 작업 시 프로젝트 플랜 파일만 읽으면 곧바로 이어서 작업을 재개할 수 있다. 
  5. 이론적으로는 매우 큰 프로젝트도 이러한 방식으로 관리가 가능하다. 
4. **쇼츠 자동 생성 성공 및 핵심 기능 완성**: 
  1. 버그 해결 후 쇼츠 생성 버튼을 누르자 드디어 정상적으로 작동한다. 
  2. 다섯 단계의 세그먼트가 생성되고, 이를 기반으로 이미지와 비디오가 자동으로 생성되어 핵심 기능이 완성된다. 
  3. 클로드 데스크톱 MCP를 활용하면 대용량 코딩, 테스트, 버그 수정, DB 관리 등 코딩의 모든 과정을 AI와 협업하여 완벽하게 처리할 수 있다. 


### 1.3. **최신 문제 해결을 위한 브라우징 MCP 활용** 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/GRtrfdSeR20/279.jpg)
1. **최신 에러 발생 상황**: 쇼츠 생성 중 에러가 발생했는데, 이는 너무 최신 문제라 클로드가 아직 해결 방법을 모르는 상황이다. 
2. **해결 방법**: 클로드에게 웹에서 직접 정보를 찾아보라고 지시한다. 
3. **브라우징 MCP 기능**: 이 기능을 통해 AI는 스스로 해결책을 찾아내며, AI가 모르는 최신 정보까지 검색하여 AI의 한계를 넘어설 수 있게 된다. 


## 2. **AI 코딩 개발 환경 설정 방법** 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/GRtrfdSeR20/346.jpg)
1. **XAMPP 설치**: 
  1. 개발 환경 구축을 위해 XAMPP를 설치한다. 
  2. 설치 옵션에서 MySQL을 반드시 함께 설치해야 한다. 
  3. XAMPP 설치 후 컴퓨터에서 웹사이트를 구동할 수 있다. 
  4. 웹사이트 기본 폴더 위치는 새 폴더로 변경하여 사용한다. 
  5. 브라우저에 'localhost'를 입력하면 개인 홈페이지가 나타난다. 
2. **클로드 데스크톱 MCP 설정**: 
  1. 이전 영상에서 다룬 MCP 설정 방법을 참고하여 진행한다. 
  2. 설정 파일을 변경하여 기존의 브라우징 기능을 새로운 MCP로 교체한다. 
  3. 새로운 MCP는 브라우저 접속, 클릭, 캡처뿐만 아니라 콘솔 로그까지 확인할 수 있게 해주어 AI가 에러를 스스로 찾아낼 수 있도록 돕는다. 
  4. 터미널 명령어 허용 목록에 MySQL을 추가하여 AI가 MySQL 데이터베이스를 다룰 수 있게 한다. 
3. **프로젝트 지침 설정**: 
  1. MCP 설치만으로는 충분하지 않으며, MCP 사용법이 담긴 프로젝트 지침도 설정해야 한다. 
  2. 프로젝트 지침에는 프로젝트 플랜 파일 관리법과 같은 핵심 노하우가 포함되어 있다. 
4. **설정 완료 확인**: 
  1. 설정이 성공적으로 완료되면 클로드 데스크톱을 재시작할 때 망치 아이콘과 숫자 42(42개 기능 활성화)가 표시된다. 
  2. 이는 AI 코딩을 위한 준비가 완료되었음을 의미한다. 
5. **노하우 및 적용 권장**: 
  1. 이 방법은 이전 영상보다 난이도가 높으므로 추가적인 노하우가 필요하며, 필요한 팁은 댓글에 자세히 제공될 예정이다. 
  2. 이 방법을 통해 AI 코딩이 훨씬 쉬워지므로 적용을 권장한다.
