## 📌 클로드 코드가 못하는 것을 코덱스 CLI가 해낼 수 있는 비결은 무엇인가?
 코덱스 cli는 GPT-5 기반으로 코드 베이스를 폭넓게 살펴보고 신중하게 문제를 해결하는 방식으로, 클로드 코드가 놓치는 <mark>어렵거나 큰 프로젝트에서 뛰어난 위력</mark>을 발휘하기 때문입니다. 


## 💡 코덱스 CLI를 효과적으로 활용하기 위한 꿀팁은?

- **모델 선택**: 계획 수립 등 어려운 작업은 '하이' 모델, 평소 코딩은 '미디움' 모델 사용.
- **어프로벌 모드**: 깊은 조사는 '리드 온리', 자유로운 코딩은 '풀 엑세스' 선택 (단, AI 실수 주의).
- **웹 검색 활성화**: config.toml 파일에 설정 추가로 웹 검색 기능 활성화.
- **WSL**** 사용**: 윈도우 환경에서는 WSL(Windows Subsystem for Linux)을 통해 설치해야 최적의 성능 발휘.

GPT-5 기반의 **코덱스 cli****(Codex CLI)**를 제대로 활용하기 위한 **숨겨진 설치 및 설정 노하우**를 공개합니다. 윈도우 사용자가 겪는 성능 저하 문제를 해결하는 **wsl****(Windows Subsystem for Linux) 활용법**부터, 기본적으로 꺼져 있는 웹 검색 기능 활성화 방법, 그리고 작업 성격에 따라 모델(Minimal, High)과 접근 모드(Read-Only, Full Access)를 전략적으로 선택하는 **최적의 AI 협업 전략**까지 모두 배울 수 있습니다. 클로드 코드(Claude Code)가 막혔던 **복잡한 ****3d 레이싱 게임**** 개발 사례**를 통해 코덱스 cli가 어떻게 코드 베이스 전체를 신중하게 분석하여 어려운 문제를 해결하는지 확인하고, 여러분의 대규모 코딩 프로젝트에 **추가 비용 없이** GPT-5의 강력한 성능을 적용해 보세요.
## 1. 코덱스 CLI의 등장 배경 및 성능 변화 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/AjlvzLbrOOg/0.jpg)
1. **개인 프로젝트 진행 중 AI 도구 사용 경험** 
  1. 화자는 3D 레이싱 게임을 개발하고 있었으며, 처음에는 재미나이 CLI를 사용했으나 막히는 부분이 많아 클로드 코드(Claude Code)로 전환했다. 
  2. 클로드 코드가 재미나이 CLI보다 훨씬 낫다고 평가했으나, 코드가 쌓이면서 클로드로도 해결하기 어려운 지점에 도달했다. 
  3. 마지막으로 코덱스 CLI(Codex CLI)를 사용한 결과, 막혔던 부분이 해결되는 놀라운 경험을 했다. 

2. **코덱스 CLI****에 대한 평가 변화 및 오해 해소** 
  1. 코덱스 CLI는 놀라운 결과를 얻기 위해 많은 노력이 필요했으며, 제대로 사용해야만 성능을 발휘한다. 
  2. 과거 코덱스 CLI는 부정적인 평가를 받았으나, 몇 주 전 **GPT-5**가 출시된 이후 평가가 긍정적으로 바뀌고 있다. 
  3. 코덱스가 오해받는 주된 이유는 **대충 세팅하면 성능이 급격히 저하**되기 때문이며, 특히 윈도우 사용자는 제대로 세팅해야 한다. 
  4. 설치 오류 시 성능이 떨어지며, 기본 설정에서 핵심 기능이 꺼져 있는 경우도 있어 저평가를 받기 쉽다. 

3. **코덱스 CLI****의 비용 효율성** 
  1. ChatGPT를 구독하고 있다면 코덱스 CLI는 **추가 비용 없이 완전 무료**로 사용할 수 있다. 
  2. GPT-5 출시 후 사용량이 늘었으며, 클로드 코드만큼 넉넉한 사용량을 제공한다. 
  3. 구독 중인 사용자는 비용 걱정 없이 코덱스 CLI를 사용해 볼 것을 권장한다. 


## 2. 코덱스 CLI 설치 및 윈도우 환경 최적화 전략 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/AjlvzLbrOOg/84.jpg)
1. **기본 설치 방법 및 준비물** 
  1. 코덱스 CLI의 설치 방법은 매우 간단하며, 특정 명령어 한 줄만 입력하면 된다. 
  2. 이 명령어가 작동하기 위해서는 **노드 JS****(Node JS)**가 컴퓨터에 반드시 설치되어 있어야 한다. 
  3. 윈도우, 맥, 리눅스 등 운영체제에 관계없이 설치 방법은 동일하다. 

2. **초기 로그인 및 비용 관리** 
  1. 설치 후 '코덱스'를 입력하면 ChatGPT 로그인 또는 API 키 입력을 요구한다. 
  2. API 키는 비용이 많이 발생할 수 있으므로 주의해야 하며, ChatGPT 로그인을 통해 구독 플랜에 따라 사용할 수 있다. 
  3. 20달러 구독 사용자 기준으로 클로드 코드와 비교했을 때 사용 시간이 밀리지 않아 코딩 분야에 대한 진심이 느껴진다. 

3. **윈도우 환경에서의 성능 최적화 (****WSL**** 활용)** 
  1. 윈도우 파워셸에서 코덱스를 설치했을 때 성능이 저하되는 문제가 발생했다. 
  2. 이는 **윈도우 네이티브 지원이 아직 실험용 버전**이기 때문이며, 제대로 된 성능을 위해서는 **WSL****(Windows Subsystem for Linux)**을 사용하는 것이 정석이다. 
  3. WSL과 윈도우 네이티브 환경의 성능 차이는 매우 크다. 
  4. WSL은 윈도우에서 리눅스를 작동시키는 기능이며, 파워셸에서 WSL install 명령어로 설치하고 WSL을 입력하여 실행할 수 있다. 
  5. WSL 환경에서 기존 설치 명령어로 코덱스를 다시 설치하면 완벽하게 작동한다. 


## 3. 클로드 코드와 코덱스 CLI의 협업 사례: 3D 레이싱 게임 디버깅 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/AjlvzLbrOOg/199.jpg)
1. **클로드 코드**** 사용 단계 및 문제 해결** 
  1. 클로드 코드가 막혔던 3D 레이싱 게임 프로젝트를 예시로 사용했다. 
  2. **초기 문제**: 게임 시작 시 자동차가 붕 떠 있고, 앞으로 이동하면 카메라만 남고 차는 사라지는 등 조작이 불가능했다. 
  3. **클로드 코드**** 작업 시작**:
  1. 다른 AI로 만들던 프로젝트를 클로드에서 이어서 작업하려면 처음에 /init을 입력해야 한다. 
  2. /init을 입력하면 클로드가 전체 코드를 분석하여 **클로드 마크다운 파일**에 결과를 저장하며, 이를 통해 클로드를 다시 시작해도 핵심 내용을 바로 파악할 수 있다. 
  4. **디버깅 전략**:
  1. 조작이 안 되는 문제를 해결하기 위해 버튼을 누를 때마다 이동 방향, 이동 거리 등의 상세 기록을 남기도록 하여 클로드가 스스로 원인을 파악하게 했다. 
  2. **플레이라이트**** ****MCP**를 활용하여 브라우저를 직접 열고 스스로 게임을 조작하며 테스트하도록 시켰다. 
  5. **클로드의 진단 및 해결**: 클로드는 차량 물리 시뮬레이션 문제, 키 입력 반응 문제 등을 찾아내어 막혔던 문제를 해결하기 시작했다. 

2. **클로드의 한계와 ****코덱스 CLI****의 투입** 
  1. 클로드 코드 사용 후 작동은 되기 시작했으나, 차가 뒤집어지거나 갑자기 멈추는 문제가 계속 발생하며 다시 막히기 시작했다. 
  2. 클로드가 물리 시스템에 인위적으로 손을 대는 문제도 발생했다. 
  3. 이 시점에 **코덱스 CLI**를 투입했다. 
  4. **코덱스 CLI**** 작업 시작**:
  1. 코덱스에도 /init 기능이 있으며, 이를 사용하면 코드 전체를 분석하여 **에이전트 마크다운 파일****(agent.md)**을 생성한다. 이 파일이 코덱스의 지침 파일 역할을 한다. 
  2. 이미 클로드 마크다운 파일이 있었기 때문에, /init을 사용하지 않고 해당 파일을 복사하여 이름만 agent.md로 변경하여 지침 파일을 순식간에 만들었다. 
  5. **코덱스의 해결 결과**: 코덱스 CLI에게 단 한 번 요청하여 테스트한 결과, 움직임이 훨씬 자연스러워졌으며, 몇 번의 추가 수정 후 제법 그럴듯한 모습이 되었다. (다만 바퀴는 여전히 이상했다.) 
  6. 이 사례는 GPT-5가 어렵거나 큰 프로젝트에서 강력한 위력을 발휘함을 보여준다. 


## 4. 코덱스 CLI의 고급 설정 및 최적화 전략 


### 4.1. 모델 선택 및 접근 모드 전략 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/AjlvzLbrOOg/357.jpg)
1. **모델 선택**** (**/model) 
  1. /model을 입력하면 **Minimal, Low, Medium, High** 모델을 선택할 수 있다. 
  2. **High 모델**: 성능이 가장 좋지만, 속도가 느리다. 
  3. **Minimal 모델**: 성능은 약하지만, 속도가 매우 빠르다. 
  4. 기본값은 중간인 **Medium**으로 설정되어 있다. 

2. **모델 활용 꿀팁** 
  1. 계획을 짤 때는 **High 모델**을 사용하고, 평소 코딩할 때는 **Medium 모델**을 사용하는 것이 효과적이다. 
  2. 정답은 없으며, 상황에 따라 적절히 모델을 바꾸는 센스가 필요하다. 

3. **접근 모드 선택 (**/approval) 
  1. 코덱스에는 **어프로벌 모드****(Approval Mode)**가 있으며, /approval을 누르면 세 가지 옵션이 나온다. 
  2. **Read-Only 모드**: 클로드 코드의 플랜 모드와 유사하며, 이름 그대로 읽기 권한만 주어져 소스 코드를 건드리지 않고 깊게 조사할 수 있다. 
  3. **Full Access 모드**: 묻지도 따지지도 않고 편집, 네트워크 접근 권한이 막힘없이 주어져 시원하게 코딩할 수 있다. 
  4. **주의사항**: AI가 실수를 자주 하고 길을 잃을 수 있으므로, 사람이 통제하지 않으면 코드가 잘못될 수 있다. 따라서 풀 액세스보다는 어느 정도 통제하는 것이 권장된다. 


### 4.2. 숨겨진 웹 검색 기능 활성화 및 파일 처리 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/AjlvzLbrOOg/454.jpg)
1. **웹 검색 기능**** 활성화** 
  1. 코덱스는 기본 상태에서 인터넷 검색(웹) 기능이 비활성화되어 있다. 
  2. 이 기능은 숨겨져 있으며, 활성화 방법은 다음과 같다. 
  1. 홈 폴더 아래 사용자 폴더로 이동한다. 
  2. .codex 폴더로 들어간다. 
  3. config.toml 파일을 찾아서 특정 내용을 추가하면 웹 검색 기능이 활성화된다. 
  3. 웹 검색이 활성화되면 구글에서 오늘 뉴스를 찾는 등 검색 아이콘이 뜨면서 웹 검색이 잘 진행된다. 
  4. 코딩 중 필요한 자료를 바로 가져올 때 유용하며, 플레이라이트로도 가능하지만 속도는 웹 검색이 가장 빠르다. 

2. **파일 처리** 
  1. 파일을 코덱스에게 보여주고 싶을 경우, 마우스로 드래그하기만 하면 된다. 
  2. 예시로 자동차 바퀴가 이상한 캡처 이미지를 끌어와 코덱스에게 보여줄 수 있다. 


## 5. MCP 설치 능력 비교 및 코덱스 CLI의 장단점 분석 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/AjlvzLbrOOg/506.jpg)
1. **MCP****(Multi-Context Processor) 설치 능력 비교** 
  1. 이전 영상에서 클로드 코드에게 MCP 설치를 자동으로 시켰던 것을 코덱스로 시도해 보았다. 
  2. 에이전트 마크다운 파일에 설치 명령을 넣어 코덱스에게 컨텍스트 7 MCP 설치를 명령했다. 
  3. 코덱스는 약 8분 뒤 설치에 성공했다. 
  4. 코덱스는 어떨 때는 잘하고 어떨 때는 헤매는 경향이 있으며, 재미나이 CLI는 이런 설치를 잘 못했다. 
  5. 코덱스 CLI는 어느 정도 해내지만, 클로드 코드만큼 능숙하지는 않았으며 어려운 MCP는 자동 설치가 쉽지 않았다. 
  6. 일단 설치만 되면 사용하는 데는 문제가 없으며, 3D 레이싱 게임 테스트 시 플레이라이트 MCP를 유용하게 사용했다. 

2. **코덱스 CLI****의 성능 만족도 및 실망 이유** 
  1. 코덱스 CLI의 기본 사용법을 훑어본 결과, 성능은 매우 만족스러웠으며 코드 리뷰 분야에서는 클로드 코드의 한계를 넘어섰다. 
  2. 사람들이 코덱스 CLI에 실망했던 가장 큰 이유는 **속도** 때문이다. 
  3. 원래 코딩 초반에는 성능이 좋지만, 코덱스 CLI는 그 좋은 구간에서조차 클로드 코드보다 느렸다. 

3. **느린 속도의 장점 (신중함)** 
  1. 코덱스의 느린 속도가 오히려 장점이 되기도 한다. 
  2. **클로드 코드**: 문제를 풀 때 딱 필요한 부분만 보며, 토큰을 아끼고 속도가 빠르지만 실수가 잦다. 
  3. **코덱스**: 하나부터 열까지 코드 베이스를 폭넓게 살펴보는 **매우 신중한 스타일**이며, 덕분에 클로드가 해결하지 못하는 어려운 일도 해낼 수 있다. 

4. **효과적인 코덱스 활용 비결 및 라이벌 구도** 
  1. 코덱스를 효과적으로 사용하는 비결은 **모델 활용**에 있다. 
  2. 리드 온리 모드나 어려운 작업에는 **High 모델**을, 간단한 작업에는 **Medium이나 Low 모델**을 사용하는 것이 좋다. 
  3. GPT가 모델을 나눈 것은 특성에 맞게 사용하라는 의도이다. 
  4. 클로드 코드에도 비슷한 기능이 있으며, **오퍼스**** 플랜 모드**에서는 느리지만 성능 좋은 오퍼스(Opus)를 사용하고, 아닐 때는 가성비 좋은 소넷(Sonnet)을 사용한다. 
  5. 예전에는 클로드 코드가 독보적이었으나, 이제 코덱스 CLI와 같은 라이벌이 등장하고 있다. 
  6. 새로운 것을 배우는 것은 피곤할 수 있지만, 그만큼 더 좋은 선택을 할 수 있게 되므로 사용자만의 최강 조합을 만들어 볼 것을 권장한다.
