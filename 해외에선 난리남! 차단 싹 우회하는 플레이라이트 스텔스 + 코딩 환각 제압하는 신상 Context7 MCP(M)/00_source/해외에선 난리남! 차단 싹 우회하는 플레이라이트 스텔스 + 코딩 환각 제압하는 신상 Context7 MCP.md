## 📌 해외 웹사이트의 AI 자동화 차단을 우회하고 코딩 시 AI 환각 현상을 제압하는 방법은?
 플레이라이트<mark> 스텔스 버전</mark>을 사용하여 자동화 차단을 우회하고, 컨텍스트 7<mark> MCP</mark>를 활용하여 AI의 코딩 환각 현상을 줄일 수 있습니다. 


## 💡 컨텍스트 7 MCP는 어떤 방식으로 AI 환각을 제압하나요?
 최신 공식 자료를 기반으로 AI에게 정확한 정보를 제공하여, AI가 헛소리(환각)를 하는 것을 줄이고 최신 api도 문제없이 다룰 수 있게 합니다. 


해외에서 화제가 되고 있는 **AI 자동화의 난제 해결법**을 총망라한 실전 가이드입니다. 웹사이트의 데이터 수집 차단을 우회하는 <strong>플레이라이트 스텔스</strong> 버전 설치부터, ai 코딩 시 발생하는 **환각(Hallucination) 현상**을 제압하는 최신 Context7 MCP 활용법까지 구체적으로 배울 수 있습니다. 이 콘텐츠를 통해 대규모 웹 스크래핑 프로젝트를 오류 없이 진행하는 **프로젝트 지침서**** 작성 노하우**와 함께, AI 자동화의 실질적인 성능과 신뢰도를 극대화하는 방법을 즉시 적용할 수 있습니다.
## 1. AI 자동화 난제 해결을 위한 새로운 접근법 소개 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Cy34dlnQKJg/0.jpg)
1. **AI 자동화 시도와 난관 발생** 
  1. 외국 팬 반응 조사를 AI로 시도하여 성공했으나, 기존 딥리서치나 MCP를 사용해도 실제 작업 시 막히는 부분이 발생함 
  2. 자동화 브라우저인 플레이라이트 사용 시 웹사이트에서 지속적으로 차단당하는 문제가 발생함 
  3. 웹사이트들이 AI에게 데이터를 뺏기지 않기 위해 민감해지고 있으며, 이러한 차단 현상은 앞으로 더욱 심해질 것으로 예상됨 

2. **난제 해결을 위한 새로운 도구 도입** 
  1. **플레이라이트**** 스텔스 버전 사용**: 자동화 작업이 들키지 않도록 스텔스 기능이 있는 브라우저를 사용하기로 결정함 
  2. **새로운 에디터 MCP 사용**: 기존 클로드 데스크탑에서 파일 쓰기 시 에러가 자주 발생하는 스트레스를 해결하기 위해 새로운 에디터 MCP를 사용함 
  3. **Context7 MCP 도입**: 해외에서 핫한 Context7을 사용하여 코딩 시 AI가 헛소리(환각) 하는 현상을 완전히 제압하고 AI의 지능을 향상시킴 

3. **실제 조사 결과 및 분석** 
  1. **조사 결과**: 손흥민 선수 해외 팬들의 반응을 국가별로 긍정/부정 의견을 조사한 결과가 도출됨 
  2. **국가별 반응**: 일본 팬들이 가장 긍정적이었고, 중국 팬들은 가장 부정적인 것으로 나타남 
  3. **분석**: 일본의 긍정적 반응은 의외였으며, 이러한 차이가 발생한 이유에 대해서도 분석을 진행함 


## 2. 프로젝트 실행 과정 및 차단 우회 방법 


### 2.1. 프로젝트 계획 및 지침서 활용 
![image]()
1. **사용 도구 및 계획**: 이전 영상에서 소개된 플레이라이트와 구글 검색 MCP를 사용했으며, 프로젝트 지침을 미리 준비함 
2. **데이터 수집**** 목표**: 다섯 개 국가를 선정하고 각 나라에서 댓글 100개씩 총 500개의 댓글을 수집하도록 계획함 
3. **프로젝트 계획 문서 활용**:
  1. 프로젝트 계획 문서를 만들어 작업이 진행될 때마다 이 문서가 계속 업데이트되도록 설정함 
  2. 이 방법을 통해 채팅창 제한이 발생해도 세 창에서 이 문서만 읽게 하여 작업을 이어갈 수 있었으며, 큰 프로젝트를 쉽게 진행할 수 있게 됨 


### 2.2. 플레이라이트 차단 문제 해결 및 작업 속도 개선 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Cy34dlnQKJg/110.jpg)
1. **초기 문제 발생**: 플레이라이트를 실행하여 실시간으로 작업을 확인했으나, 유명 사이트들이 플레이라이트를 계속 차단하여 작업 중간에 에러가 발생함 
2. **스텔스 버전 도입**: 해결책으로 **플레이라이트**** 스텔스 버전**을 설치했고, 이를 사용하자 에러가 사라짐 
3. **데이터 수집**** 성공**: 스텔스 버전 덕분에 일본어와 중국어 댓글도 잘 보였으며, AI가 댓글을 열심히 수집함 
4. **텍스트 에디터 MCP**** 활용**:
  1. 중간에 보이는 텍스트 에디터는 이전 영상의 파일 시스템 MCP 대신 **텍스트 에디터 MCP**로 파일을 쓰는 데 사용됨 
  2. 이로 인해 에러가 확 줄고 작업 속도가 빨라짐 
5. **자동화 진행**: 클로드 데스크탑은 한 번만 동의하면 계속 자동으로 진행되어 작업이 빠르게 끝났고, 데이터 폴더에 댓글 파일들이 채워짐 


### 2.3. 최종 결과물 제작 및 웹 검색 최적화 전략 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Cy34dlnQKJg/156.jpg)
1. **결과물 제작**: 수집된 자료를 웹페이지(HTML)로 만들어 달라고 요청함 
2. **토큰 제한**** 대응**: 큰 작업은 토큰 제한 때문에 세 창으로 진행하는 것이 좋으며, 세 창에서 프로젝트 플랜 파일을 먼저 읽으면 쉽게 작업을 이어서 진행할 수 있음 
3. **HTML 제작**** 및 ****디버깅**: HTML 제작은 약 5분 만에 완료되었으며, 중간에 에러가 발생하자 AI가 스스로 디버깅을 하도록 시켜 스크린샷을 찍고 코드를 빠르게 수정함 
4. **최종 완성 및 속도 개선**: 최종 결과물이 완성되었고, 작업 시간이 이전보다 훨씬 빨라짐 
5. **웹 검색 최적화 권장 사항**:
  1. 웹 검색은 **1차 ****구글 검색**** 후 2차 웹브라우징**을 꼭 권장함 
  2. 웹브라우징만으로 진행하면 한 사이트 내에서 검색하고 클릭하는 과정이 너무 느리고 오류도 많기 때문임 
  3. 구글 검색으로 결과 페이지에 바로 들어가면 속도가 빠르고 품질도 좋아지는 효과를 얻을 수 있음 


## 3. Context7 MCP를 활용한 코딩 환각 제압 및 설치 방법 


### 3.1. Context7 MCP의 효과 및 신뢰도 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Cy34dlnQKJg/200.jpg)
1. **Context7 MCP 소개**: 해외 개발자 사이에서 핫한 Context7 MCP 사용법을 소개함 
2. **실제 문제 해결 사례**: 실제 사례로 인덱스 HTML 파일에서 웹 파일 로드가 안 되어 클로드가 헤매는 상황이 발생했을 때, Context7을 사용하자 정확하고 깔끔한 해결법을 제시해 줌 
3. **제시된 해결책**: 웹서버 실행, 브라우저 설정 변경, 대체 기술 사용 등 명확한 방법과 그 이유까지 완벽히 설명해 줌 
4. **환각 현상 감소**: Context7은 최신의 공식 자료를 사용하여 클로드의 **환각 현상(Hallucination)**을 줄이고 최신 API도 문제없이 커버하여 신뢰도가 매우 높음 
5. **중요성**: 환각 증상은 코딩 에러의 가장 대표적인 증상이기 때문에 Context7 사용을 놓치지 않도록 추천함 


### 3.2. Context7 및 필수 MCP 설치 및 설정 방법 
![image]()
1. **Context7 ****설정 파일 편집**:
  1. 클로드에서 파일 설정 > 개발자로 들어가 설정 편집을 누름 
  2. 설정 파일을 메모장으로 열고 제공된 설정 내용을 그대로 복사하여 붙여넣으면 됨 
2. **Context7 ****API**** 키 발급**: API 키는 공식 사이트에서 무료로 발급받을 수 있으며, 일정 사용량 초과 시 유료이지만 기본적으로는 무료 사용이 가능함 
3. **로컬 MCP 설치**: 텍스트 에디터와 플레이라이트 스텔스 MCP는 로컬에 설치하면 훨씬 편리함 
4. **명령어 실행**: 명령 프롬프트에서 다음 두 명령을 실행하여 설치함 
5. **적용 및 재시작**: 설치와 설정 파일 편집이 끝났다면 클로드 또는 컴퓨터를 재시작하면 정상 적용됨 
6. **기존 버전 처리**: 기존에 일반 플레이라이트 버전이 설치되어 있다면 에러가 발생할 수 있으므로, 이를 지우고 스텔스 버전을 설치해야 함 (삭제 방법은 댓글에서 확인 가능) 


### 3.3. 추가 권장 사항 및 정보 공유 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Cy34dlnQKJg/286.jpg)
1. **프로젝트 지침서**** 제공**: 프로젝트 지침서도 함께 제공될 예정임 
2. **MCP 사용 권장**: 파일 시스템 MCP 대신 **텍스트 에디터 MCP** 사용을 권장하며, 이는 파일 쓰기 오류를 많이 줄여줌 
3. **피드백 요청**: 이 내용이 도움이 되었거나 어려움이 있는 경우 댓글로 알려주면, 아는 정보를 열심히 공유하겠다고 밝힘
