## 📌 Gemini CLI를 '개조'하여 어떤 개선 효과를 얻었는가?
 Gemini CLI에 <mark>MCP(모델 컨텍스트 프로토콜) 중 '시퀀셜 싱킹'을 설치</mark>하여 AI의 지능을 높이고, 플래시 모델로도 프로 모델에 근접하는 성능을 무료로 활용할 수 있게 되었습니다. 


## 💡 시퀀셜 싱킹 MCP 설치 후 어떤 변화가 있었는가?
 테트리스 게임 생성 시 <mark>초기 오류 수정 횟수가 현저히 줄어들고</mark>, 한 번에 더 완성도 높은 코드를 생성하는 등 AI의 문제 해결 능력이 향상되었습니다. 


Gemini CLI의 **무료 사용 한계를 극복**하고 AI의 지능을 획기적으로 높이는 실용적인 방법을 공개합니다. 엔트로픽의 통신 규칙인 **MCP(모델 컨텍스트 프로토콜)**를 Gemini CLI에 설치하여, 무료 모델인 'Flash'로도 유료 모델 'Pro'에 근접하는 **단계적 사고(Sequential Thinking)** 능력을 구현하는 구체적인 과정을 배울 수 있습니다. 이 콘텐츠를 통해 AI 코딩 결과물의 품질을 대폭 개선하고, 무료 AI 자원을 최대한 활용하는 **고급 AI 활용 전략**을 습득할 수 있습니다.
## 1. Gemini CLI의 무료 사용 한계 및 MCP 도입 배경 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/_CUZaKgvPJI/2.jpg)
1. **Gemini CLI의 기본 정보 및 무료 사용 조건** 
  1. Gemini CLI는 Gemini AI에서 제공하는 커맨드 라인 인터페이스 AI이다. 
  2. 현재 (영상 제작 시점 기준) 올해까지 무료로 사용할 수 있다. 
  3. 무료 사용 조건은 다음과 같다. 
  1. 하루 최대 <mark>1천 회</mark> 요청
  2. 분당 <mark>60회</mark> 요청
  3. <mark>100만 토큰</mark> 컨텍스트까지 사용 가능
  4. 사용 내역은 몇 시간마다 재충전된다.
  4. 유료 모델인 'Pro' 사용량이 다 되는 경우가 있어, 예전에 무료 모델인 'Flash'를 사용할 수 있는 명령어를 전달한 적이 있다. 

2. **Flash 모델의 성능 한계 및 개선 필요성** 
  1. Flash 모델을 사용하여 코딩(바이브 코딩)을 진행하면 <mark>거의 쓸 수 없는 내용</mark>이 나오는 경향이 있다. 
  2. 정확한 위치 지정 및 함수 변경 지정을 하는 경우에는 문제가 생기지 않지만, 사용자들이 '전부 해주는' 세상을 기대하고 있으므로, 이번 영상에서는 Flash 모델로도 **전부 해주는** 결과를 얻는 방법을 시도한다. 
  3. **오늘의 목표**: Gemini CLI에서 Flash 모델을 이용해 테트리스 게임을 만들어 달라고 요청하는 것이다. 


## 2. Flash 모델을 이용한 테트리스 제작 1차 시도 (MCP 미적용) 
![image]()
1. **Flash 모드 설정 및 상세 요청 사항 입력** 
  1. Flash 모드로 동작하도록 설정한다. 
  2. 테트리스 게임 제작을 위해 다음과 같은 상세 요청 사항을 입력한다. 
  1. 현재 폴더에 테트리스 게임을 할 수 있는 <mark>HTML 파일</mark>을 생성해 줄 것
  2. 가급적 현대적으로 보일 수 있도록 잘 꾸며주고 <mark>방향키</mark>를 사용해 조작할 수 있게 해 줄 것
  3. 점수 체계는 다음과 같이 구현할 것:
  1. 한 번에 하나 터질 때: <mark>10점</mark>
  2. 두 개 터질 때: <mark>30점</mark>
  3. 세 개 터질 때: <mark>60점</mark>
  4. 네 개 터질 때: <mark>100점</mark>
  4. <mark>1천 점</mark>에 도달할 때마다 블록이 자동으로 내려가는 속도가 증가하는 형식으로 게임을 구현할 것
  5. 애매한 부분이 있으면 다시 물어보고 진행하도록 요청한다.
  3. 항상 신뢰를 선택해서 진행하도록 한다. 

2. **1차 결과 확인 및 오류 수정 과정** 
  1. 게임이 완성되었다는 메시지가 나왔으나, 실행 결과 <mark>아무것도 안 나오고 오류</mark>가 발생했다. 
  2. **오류 수정 요청**: 블록이 생성되어 내려오지 않는 문제를 수정해 달라고 요청했다. 
  3. 정상적으로 수정되지 않아 <mark>여러 번 더 수정</mark>을 요청하는 상황이 발생했다. 
  4. 여러 번의 수정 끝에 '게임 스타트' 버튼을 누르자 제대로 실행되었다. 

3. **기능 테스트 및 토큰 사용량 확인** 
  1. **점수 기능 테스트**: 하나 터졌을 때 정상적으로 <mark>10점</mark>이 올라가는 것을 확인했다. 
  2. 두 칸이 한 번에 터졌을 때 <mark>30점</mark>이 올라가는 것을 확인했다. 
  3. 이 과정에서 **굉장히 많은 수정 사항**이 발생했음을 확인할 수 있다. 
  4. 컨텍스트 레프트(Context Left)가 <mark>96%</mark> 남은 것으로 보아, 1차 시도 과정에서 컨텍스트를 <mark>약 4%</mark> 소모했음을 알 수 있다. 


## 3. MCP(모델 컨텍스트 프로토콜) 설치 및 Sequential Thinking 적용 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/_CUZaKgvPJI/194.jpg)
1. **MCP 개념 설명** 
  1. MCP는 **모델 컨텍스트 프로토콜(Model Context Protocol)**의 약자로, 엔트로픽에서 만든 통신 규칙이다. 
  2. 통신 규칙이란 핸드폰에서 전화번호를 누르면 상대방에게 연결되는 것과 같은 일종의 통신 방식이다. 
  3. Gemini CLI에 MCP를 설치하여 AI의 지능을 높이는 것이 목표이다. 

2. **Sequential Thinking MCP 설치 과정** 
  1. **목표**: Gemini CLI에 **시퀀셜 싱킹(Sequential Thinking)**이라는 MCP를 설치하여 AI의 지능을 높여준다. 
  2. 이 MCP를 설치하면 AI가 <mark>단계적으로 생각</mark>하여 여러 내용을 처리하므로, Flash 모델도 거의 Pro 모델에 근접하는 실력을 보여줄 수 있다. 
  3. **설치 방법**: 스미더리 AI(Smither AI)를 이용한다. 
  1. 검색창에 '시퀀셜 싱킹'을 검색하여 스미더리 AI에 만들어 둔 것을 선택한다. 
  2. 오토(Auto)에서 Gemini CLI를 설치할 것을 선택하고, 로그인을 진행한 후 커넥트 버튼을 눌러 연결한다. 
  3. 오른쪽에 나오는 명령줄을 카피 버튼을 눌러 복사한 뒤 붙여 넣기를 실행하여 설치를 완료한다. 
  4. **설치 확인**: Gemini CLI로 돌아가서 /MCP list 명령어를 사용하여 MCP가 정상적으로 설치되었는지 확인한다. 
  5. 시퀀셜 싱킹 MCP가 정상적으로 설치된 것을 확인했다. 

3. **MCP 설치의 다른 방법들** 
  1. 명령줄이나 클로드 코드를 사용하여 Gemini에 설치를 요청하거나, 직접 JSON 파일로 만들어서 삽입하는 등 여러 가지 방법이 있다. 


## 4. MCP 적용 후 테트리스 제작 2차 시도 및 결과 비교 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/_CUZaKgvPJI/303.jpg)
1. **2차 시도 요청 및 프롬프트 설정** 
  1. 아까와 동일하게 테트리스 게임을 만들어 달라고 요청한다. 
  2. 프롬프트 고도화를 심하게 하면 결과가 달라질 수 있으므로, 속이지 않기 위해 <mark>비슷하게 요청</mark>하되 **추가적으로 MCP만 더 사용**하라고 명령을 내린다. 

2. **2차 시도 결과 확인** 
  1. 완료까지 대략 <mark>20분 조금 넘게</mark> 걸렸다. 
  2. 실행 결과, 아까와는 모양이 좀 다르게 나왔다. 
  3. '스타트 게임' 버튼을 누르자 작동이 정상적으로 되었다. 
  4. **점수 기능 테스트**: 한 번에 두 칸이 터졌을 때 정상적으로 <mark>30점</mark>이 올라가는 것을 확인했다. 
  5. 한 번에 나온 결과치고는 생각보다 괜찮은 결과가 나왔으며, 여러 번 수정한 1차 시도에 비해 훨씬 편하게 만들었다고 평가된다. 


## 5. MCP 활용의 장점과 한계 
![image]()
1. **MCP 활용의 장점** 
  1. 가장 큰 장점은 <mark>무료로 조금 더 나은 AI</mark>를 사용할 수 있다는 점이다. 
  2. 시퀀셜 싱킹 MCP를 사용하면 토큰 사용량이 급증하는 문제가 있지만, 무료라는 점이 이 문제를 거의 다 커버해 줄 수 있다. 

2. **MCP 활용의 한계점** 
  1. 클로드 등 <mark>토큰 비용이 비싼 상태</mark>에서는 사용하기 어렵다. 
  2. 현재 MCP 연결을 지원하지 않고 <mark>API 추가만 지원하는 AI</mark>의 경우에는 MCP를 이용할 수 없다. 
  3. 이 방법의 활용도는 Gemini에게 달아 사용하는 방식이 거의 한계일 것으로 예상된다. 
  4. Gemini에서도 <mark>모든 MCP를 다 지원하는 것은 아니므로</mark>, 사용자가 하나하나 붙여보면서 테스트를 진행해 보아야 한다. 

3. **결론**: Gemini Pro를 많이 사용하지 못해 아쉬워하는 사용자들에게 도움이 되고자 이번 영상을 준비했다.
