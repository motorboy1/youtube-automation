## 1. AI 코딩의 문제점과 획기적인 AI 협업 시스템 소개 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/BG8KMv285CI/0.jpg)
1. **기존 AI 코딩의 문제점 인식** 
  1. AI가 코드를 빠르게 생성하지만, 에러가 폭발하여 **무한 반복의 늪**에 빠지는 경우가 많다 
  2. 높은 토큰 비용 문제도 심각하여 월 20달러로는 부족하고, 결국 100달러, 200달러까지 지출하게 된다 

2. **문제 해결을 위한 ****AI 협업**** 시스템 구축** 
  1. 이 시스템은 기존의 **무한 에러 문제**와 **높은 토큰 비용 문제** 두 가지를 획기적으로 해결한다 
  2. ChatGPT의 **딥리서치**를 통한 정밀 분석 능력과 클로드의 **MCP(Mission Control Panel)** 기반 자동화 코딩 능력을 결합하여 100달러 이상의 퍼포먼스를 낸다 

3. **AI 협업**** 시스템의 작동 방식 (사례)** 
  1. **문제 발생**: 바이브 코딩으로 만든 프로그램에서 쇼츠 생성 버튼을 눌렀을 때 에러가 발생한다 
  2. **ChatGPT의 분석**: 똑똑한 ChatGPT에게 원인을 문의하여 해결책과 코드 예제를 확실히 받아낸다 
  3. **클로드로 전달 및 적용**: ChatGPT의 결과를 클로드에게 전달하며, 한 번 더 점검하고 바로 적용하라고 지시한다 
  4. **자동 작업 완료**: 클로드가 지시에 따라 작업을 자동으로 완료하고 에러를 해결한다 
  5. **효율성 증대**: 예전에는 몇 시간이 걸리던 작업이 이제 몇 분이면 끝나며, 정확도가 올라가 작업이 훨씬 편해진다 

4. **클로드 MCP**** 활용 및 복잡한 프로젝트 적용** 
  1. 클로드가 다르게 움직이는 것은 여러 **MCP(Mission Control Panel)**를 활용하여 효율을 극대화했기 때문이다 
  2. 이 시스템은 복잡한 오토 블로그의 쇼츠 기능까지 추가한 프로젝트에 적용되었으며, 소스가 복잡해지면서 에러가 잦아지고 무한 에러 지옥에 빠지는 문제가 발생했었다 
  3. AI 협업 시스템을 통해 바이브 코딩의 문제점을 해결하고 에러가 줄어 결과물이 빨리 나오게 되었다 
  4. **비용 절감**** 효과**: 클로드 20달러 요금제를 사용할 때 예전에는 한계에 부딪혀 다섯 시간씩 기다려야 했으나, ChatGPT 및 제미나이와 협업하면서 정확도가 올라가고 코딩량이 줄어들어 20달러로도 작업이 매끄럽게 잘 된다 
  5. 결과적으로 적은 코딩량으로 최대 효과를 거두어 비용 절약과 효율성 두 마리 토끼를 모두 잡았다 

5. **협업의 핵심 비결: GitHub 중심 방식** 
  1. 이 시스템의 비결은 **깃허브****(GitHub)**를 중심으로 하는 방식이다 
  2. 깃허브를 중간에 두어 클로드가 만든 코드를 ChatGPT가 이어서 작업하고, 제미나이까지 이어서 작업할 수 있는 **진짜 AI 간의 협업**이 가능해진다 
  3. 이 방식은 각 AI의 장점만 쏙쏙 뽑아서 진행하는 획기적인 방법이다 


## 2. 딥리서치(ChatGPT)를 활용한 정밀 분석 및 클로드와의 연동 과정 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/BG8KMv285CI/172.jpg)
1. **문제 발생 및 ****딥리서치**** 활용 결정** 
  1. 쇼츠 생성하기를 눌렀을 때 빨간 에러 팝업이 떴으나, 사실은 제미나이 키가 제대로 있는 **잘못된 오류**였다 
  2. 원래는 클로드에게 맡겼을 상황이지만, 이번에는 뭔가 놓친 느낌이 강하여 **딥리서치**를 활용하기로 결정했다 
  3. 딥리서치는 조금 오래 걸리지만 **정확도가 끝내주게 꼼꼼하게 분석**하는 장점이 있다 
  4. 딥리서치와 클로드, ChatGPT가 연결될 수 있는 비밀 통로는 바로 **깃허브**였다 

2. **GitHub 기반 ****AI 협업**** 환경 구축** 
  1. **준비 단계**: 깃 MCP를 설치하고 깃허브 CLI까지 설치하여 준비를 완료했다 
  2. 클로드 데스크탑 터미널 MCP와 합쳐서 깃허브 관리를 자유롭게 할 수 있게 되었다 
  3. **꿀팁**: 깃허브는 무한 복제가 가능하므로, 깃허브 저장소를 만들고 클로드 데스크탑에 로컬 소스를 복제하여 깃허브 저장소와 클로드가 완전히 똑같은 소스를 가지게 했다 
  4. 이 깃허브 저장소를 ChatGPT 딥리서치와 연결했다 

3. **ChatGPT ****딥리서치****를 통한 오류 분석** 
  1. ChatGPT에게 에러 메시지를 설명하고 원인과 해결책, 그리고 소스 예제까지 요청했다 
  2. 분석에 약 10분 정도 시간이 걸렸지만, 결과는 **정확하고 꼼꼼했다** 
  3. 딥리서치 결과를 **PDF 파일**로 다운로드했다 

4. **클로드의 코드 수정 및 검증 과정** 
  1. 다운로드한 PDF 파일을 클로드 데스크탑에 전달하고, PDF를 보고 수정하되 **한 번 더 검증**해 보라고 요청하여 만전을 기했다 
  2. **토큰 절약 전략**: 딥리서치로 이미 꼼꼼하게 분석했으므로, 토큰 소모가 큰 테스크 매니저 MCP인 **슈림프**** 없이** 작업을 진행하도록 요청했다 
  3. 클로드는 PDF대로 작업을 수행하고 스스로 검증까지 진행했으며, 이 과정에서 **더 좋은 해결책**을 찾아내기도 했다 
  4. **작업 완료 및 커밋**: 작업이 완료되면 깃 커밋(Git Commit)을 수행하는데, 이는 게임의 세이브 포인트와 같아서 나중에 문제가 생기면 이 지점으로 언제든지 돌아올 수 있다 
  5. 테스트 결과 에러가 완벽히 해결되었으며, 보통 복잡한 소스는 에러 해결에 시간이 꽤 걸리지만 이번에는 **20분**이면 충분했다 


## 3. GitHub 브랜치 운영 전략 및 제미나이 연결 방법 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/BG8KMv285CI/307.jpg)
1. **Git ****브랜치**** 운영 전략 (****평행 우주**** 비유)** 
  1. 원래 있던 소스에 클로드의 깃 MCP 도움을 받아 깃을 추가하고 브랜치를 만들었다 
  2. **브랜치**** 개념**: 깃 브랜치는 영화의 평행 우주와 비슷하며, 두 개의 세상(브랜치)을 만들었다 
  1. **테스트 우주 (Test Branch)**: 마음껏 실험해 보는 공간이다 
  2. **메인 우주 (Main Branch)**: 깨끗한 원본 소스가 있는 공간이다 
  3. **작업 흐름**: 테스트 우주에서 실험하고 결과가 좋으면 메인 우주와 합치며, 테스트 우주가 엉망이 되어도 깨끗한 메인 우주에서 새로운 테스트 우주를 만들 수 있다 
  4. 작업이 성공하면 테스트 우주를 메인 우주와 병합하고 이를 깃으로 올려서 클로드, ChatGPT, 제미나이까지 같은 세상(브랜치)을 쉽게 공유할 수 있다 
  5. 깃 MCP 덕분에 이 모든 작업을 클로드에게 **말로만 지시**하여 끝낼 수 있었다 

2. **제미나이(Gemini)와 GitHub 연결 방법** 
  1. 제미나이에서 파일 추가를 누르고 **코드 가져오기**를 선택한다 
  2. 깃허브 주소를 넣고 가져오기를 클릭하면 연결 여부를 물어본다 
  3. **연결 실패 및 해결**: 깃허브 저장소가 비공개로 설정되어 있어 연결에 실패한다 
  4. "깃허브에 연결해 보세요"를 눌러 권한 확인 절차를 진행하고, 저장소를 선택한 뒤 "인스톨"을 눌러 권한 설정을 완료한다 
  5. 다시 "가져오기"를 누르면 깃허브 저장소가 성공적으로 연결된다 
  6. 제미나이에게 필요한 내용을 요청하면 원인과 해결법, 코드까지 알려주며, 그 코드를 복사하여 클로드에게 전달하면 된다 

3. **AI 선택 및 최종 워크플로우 정리** 
  1. ChatGPT의 딥리서치나 제미나이 중 원하는 것을 선택하여 사용하면 된다 
  2. **정확성 비교**: 딥리서치가 더 오랫동안 고민하므로 정확성이 더 좋을 것으로 추정된다 
  3. **아쉬운 점**: 구글의 제미나이는 딥리서치에서 깃허브 연결이 안 되는 점이 아쉬웠다 
  4. **필수 세팅**: 깃허브를 연결하려면 로컬 컴퓨터에 **깃 설치**가 필수이며, 작업 폴더를 깃허브와 연결해야 코딩 후 바로 소스 업로드가 가능하다 
  5. **ChatGPT 연결**: ChatGPT도 커넥터라는 기능으로 깃허브 연결이 가능하다 
  6. **최종 결론**: 이 방법을 활용하면 클로드, ChatGPT, 제미나이를 함께 사용하여 **개발 기간 단축**은 물론, 토큰 사용량 최적화를 통한 **비용 절약**까지 가능하다 
  7. 딥리서치의 정확함과 클로드 MCP의 편리함을 섞은 이 조합은 개발 효율을 극대화하는 꿀 조합이다
