## 2. 무료 웹사이트 호스팅 및 FTP 연결 설정 


### 2.1. 무료 홈페이지 호스팅 신청 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/86.jpg)
1. **닷홈 ****무료 호스팅**** 신청 절차** 
  1. 닷홈에서 무료 홈페이지 호스팅을 신청해야 한다. 
  2. 로그인 후 '무료 호스팅 신청하기'를 클릭한다. 
2. **무료 호스팅****의 조건 및 정보 저장** 
  1. 무료 호스팅은 6개월 동안 사용 가능하며, 연장도 가능하여 무기한 무료 사용이 가능하다. 
  2. FTP 아이디, 비밀번호, DB명, DBID, DB 비밀번호, PHP 버전을 반드시 메모해 두어야 한다. 
  3. 신청 완료 후 제공되는 홈페이지 주소와 IP 정보도 함께 기록해야 한다. 


### 2.2. 레이드라이브 설치 및 계정 연결 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/117.jpg)
1. **레이드라이브****의 역할 및 설치** 
  1. 레이드라이브는 FTP 서버를 내 컴퓨터 폴더처럼 쉽게 사용할 수 있도록 돕는 무료 프로그램이다. 
  2. 구글에서 '레이드라이브'를 검색하여 다운로드하고 설치를 완료한다. 
  3. 설치 후 프로그램을 실행하고 구글 계정으로 로그인하면 편리하다. 
2. **레이드라이브****를 이용한 홈페이지 계정 연결** 
  1. 레이드라이브에서 '추가'를 클릭한 후 'NAS'를 선택하고 'FTP'를 누른다. 
  2. 메모해 둔 홈페이지 IP, FTP 아이디, 비밀번호를 입력하고 '연결' 버튼을 누른다. 
  3. 연결이 완료되면 컴퓨터에 Z드라이브가 생성되며, Z드라이브 내의 HTML 폴더가 홈페이지 파일이 있는 위치이다. 
  4. 클로드가 이 폴더에 파일을 생성하면 홈페이지에 즉시 반영되어 코딩과 배포가 동시에 이루어진다. 


## 3. 클로드 데스크톱 및 MCP(Multi Agent Conversation Program) 설정 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/170.jpg)
1. **클로드 데스크톱****의 ****프로젝트 지침**** 및 MCP 활용** 
  1. 클로드 데스크톱의 프로젝트에 지침(프롬프트)을 입력하여 작업을 지시한다. 
  2. MCP(Multi Agent Conversation Program)라는 도구를 활용하는데, 최근 업데이트 이후 망치 아이콘이 아닌 툴 아이콘에서 찾을 수 있다. 
  3. MCP에서 원하는 기능을 켜거나 끌 수 있다. 
2. **주요 MCP 기능 설명** 
  1. **컨텍스트**: 코딩 오류를 방지하는 역할을 한다. 
  2. **텍스트 에디터**: 파일 시스템 MCP의 고질적인 에러를 해결하고 토큰을 절약해 준다. 
  3. **구글 서치**: 구글에서 자료를 직접 검색한다. 
  4. **플레이라이트 스텔스**: 접근하기 어려운 웹페이지도 잘 보여준다. 
  5. **터미널**: 컴퓨터 명령어를 실행할 수 있다. 
  6. **이미지 생성**: 최신 GPT-4 모델로 이미지를 자동으로 생성하여 품질이 뛰어나다. 
  7. MCP에 대한 자세한 내용은 이전 영상을 참고해야 한다. 
3. **텍스트 에디터의 주요 이점** 
  1. 텍스트 에디터는 Z드라이브에 파일을 직접 생성하며, 이 덕분에 파일이 만들어지는 즉시 닷홈 홈페이지에 반영된다. 
  2. 이는 시간을 절약해 주는 매우 편리한 기능이다. 


## 4. 클로드를 활용한 웹사이트 개발 과정 


### 4.1. 프로젝트 계획 및 초기 설정 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/237.jpg)
1. **관광 사이트 리서치 및 이미지 생성 계획** 
  1. 제주도 관광 홈페이지 제작을 위해 먼저 관광 사이트를 리서치한다. 
  2. 구글 검색과 플레이라이트 스텔스 MCP를 사용하여 리서치하고, 결과는 문서로 저장된다. 
  3. 클로드는 GPT-4<mark> 이미지 모델 MCP</mark>를 사용하여 멋진 이미지를 직접 생성한다. 
  4. 웹에서 사용될 이미지 크기가 정해져 있으므로, 클로드가 필요한 크기로 이미지를 잘라주는 프로그램을 직접 만들도록 지시하여 필요할 때 호출해 사용하도록 한다. 
2. **프로젝트 계획 문서 작성 및 파일 저장 방식** 
  1. 전체 과정을 관리하고 작업 진행 상황을 정리하기 위해 프로젝트 계획 문서를 만든다. 
  2. 이 문서는 채팅창 제한이나 에러 발생 시에도 작업을 이어갈 수 있도록 돕는다. 
  3. 파일 저장은 Z드라이브를 통해 웹 호스팅과 연결된 곳에 클로드가 직접 접근하여 생성, 수정, 삭제하도록 한다. 
  4. 이는 터미널과 텍스트 에디터 MCP를 통해 가능하며, 클로드가 파일을 생성하자마자 바로 웹에 배포되는 방식이다. 


### 4.2. 웹 리서치부터 프런트엔드/백엔드 개발 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/318.jpg)
1. **유명 관광 웹사이트 분석** 
  1. 플레이라이트 스텔스 MCP가 브라우저를 통해 유명 관광 웹사이트를 분석한다. 
  2. 이 MCP는 페이지 분석, 코드 체크, 스크린샷 촬영뿐만 아니라 콘솔 로그로 오류 체크까지 하여 혼자 디버깅할 수 있다. 
2. **상세 기획 및 페이지 생성** 
  1. 웹 리서치가 끝나면 상세한 기획이 시작된다. 
  2. HTML, CSS, 자바스크립트를 이용해 메인 페이지를 직접 만들며, 필요한 이미지를 스스로 생성한다. 
  3. 이미지 생성 프롬프트를 만들고, 이미지를 직접 생성하여 적절한 위치에 저장한다. 
  4. 백엔드인 PHP 코드 작성 및 데이터베이스 작업도 가능하다. 
  5. 닷홈 무료 홈페이지 신청 시 메모해 둔 DB 정보와 PHP 버전을 프로젝트 지침에 넣어두면, 클로드가 이를 기반으로 DB 작업과 PHP 코드를 스스로 생성한다. 
  6. 기획부터 디자인, 프런트엔드, 백엔드, 배포까지 전 과정이 풀자동 에이전트로 진행된다. 


### 4.3. 효율적인 파일 처리 및 디버깅 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/387.jpg)
1. **토큰 절약을 위한 파일 처리 방식** 
  1. 클로드가 웹페이지 작업을 지속하며, 텍스트 에디터는 파일 전체가 아닌 20번째 줄처럼 <em>꼭 필요한 부분만 읽는다</em>. 
  2. 이는 토큰을 절약하여 더 많은 작업을 할 수 있게 하며, 커서 AI의 시스템 방식을 텍스트 에디터 MCP가 구현한 것이다. 
2. **오류 방지 및 작업 연속성 확보** 
  1. 파일 생성 시 한 번에 너무 길면 에러가 발생할 수 있으므로, 처음에는 조금씩 만들고 나중에 추가하도록 지침에 설정한다. 
  2. 이 방식은 에러가 나도 작업을 이어붙일 수 있도록 한다. 
3. **웹 브라우저에서 결과 확인 및 이미지 저작권** 
  1. 웹 브라우저에서 도메인만 입력하면 완성된 홈페이지를 세계 어디에서든 확인할 수 있다. 
  2. 홈페이지의 이미지들은 클로드가 직접 만든 것이므로 저작권 걱정이 없다. 
4. **클로드의 자체 ****디버깅**** 및 디자인 수정 능력** 
  1. 교통 정보 페이지 작업 중 디자인 이상 발견 시, 클로드에게 화면을 캡처하게 하여 스스로 문제점을 찾도록 지시한다. 
  2. 클로드는 스타일 충돌과 배치 이상을 스스로 결론 내리고 코드를 수정하기 시작한다. 
  3. 말로 설명하기 어려운 디자인 문제를 스스로 확인하고 수정하여 버그를 해결하며, 디자인을 깔끔하게 개선한다. 
  4. GPT-4의 이미지 퀄리티는 매우 뛰어나다. 


### 4.4. 페이지 완성 및 기능 추가 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/471.jpg)
1. **여행팁 페이지 이미지 자동 생성 및 완성** 
  1. 여행팁 메뉴에 이미지가 빠져 있을 경우, 필요한 이미지를 만들도록 지시하면 클로드가 여러 이미지를 한 번에 만들어 적절한 위치에 자동 저장한다. 
  2. 이를 통해 보기 좋은 여행팁 페이지가 완성된다. 
2. **관광지 페이지의 DB 및 ****PHP**** 연결** 
  1. 초기 HTML로만 구성된 관광지 페이지를 클로드가 데이터를 DB에 넣고 PHP로 연결한다. 
  2. 이를 통해 검색이나 필터링 기능이 가능해진다. 
  3. 클로드가 파일을 직접 만들어 필요한 내용을 DB에 넣고 혼자서 처리하여 검색 필터링 기능까지 완벽하게 작동한다. 
3. **홈페이지의 점진적 완성** 
  1. 클로드는 각 페이지를 점진적으로 발전시키며 홈페이지를 완성해 나간다. 


## 5. 완성된 웹사이트의 특징 및 MCP 설치 가이드 


### 5.1. 완성된 홈페이지의 품질과 경제성 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/509.jpg)
1. **홈페이지 품질 및 기능** 
  1. 클로드로 완성된 제주도 관광 홈페이지는 맞춤형 홈페이지 제작비가 수백만 원에 달하는 수준의 품질을 초고속으로 달성한다. 
  2. 단순 디자인을 넘어 실제 프로그램 기능까지 작동한다. 
2. **데이터베이스**** 확인 및 유지보수 이점** 
  1. 닷홈을 통해 데이터베이스 조회 페이지로 들어가면 실제로 테이블과 데이터가 존재함을 확인할 수 있다. 
  2. DB에 한 줄을 추가하면 홈페이지에 바로 반영된다. 
  3. 관리자 페이지도 쉽게 추가할 수 있으며, 이렇게 만든 웹사이트는 유지비가 거의 들지 않아 매우 경제적이다. 


### 5.2. MCP 설치 및 프로젝트 지침의 중요성 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UGZTzPS2VZU/550.jpg)
1. **MCP 사용을 위한 필수 사전 설치** 
  1. MCP를 사용하기 위해서는 Node.js와 Python이 필요하며, 공식 홈페이지에서 다운로드하여 설치해야 한다. 
2. **MCP 설치 및 설정** 
  1. MCP는 윈도우의 경우 명령 프롬프트에서 미리 설치해야 하며, 설치 명령어는 댓글로 제공된다. 
  2. 설치 후 설정 내용을 클로드 데스크톱 컨피그 파일에 붙여 넣어야 한다. 
  3. API 키를 발급받아 설정 파일에 입력해야 하며, 발급 방법은 댓글을 참고해야 한다. 
3. **프로젝트 지침****의 핵심 역할** 
  1. 프로젝트 지침은 매우 중요하며, 특히 이미지 생성 MCP는 최신 기능이므로 클로드가 혼란스러워할 수 있어 지침서가 필수적이다. 
  2. 지침서에는 홈페이지 도메인, 소스 폴더 위치, DB 정보, PHP 사용 여부 등이 포함된다. 
  3. 텍스트 에디터 MCP 사용 시 자주 발생하는 오류 방지법도 지침서에 포함되어 있어 시행착오를 줄일 수 있다. 
  4. 이 모든 노하우가 지침서에 담겨 있다.
