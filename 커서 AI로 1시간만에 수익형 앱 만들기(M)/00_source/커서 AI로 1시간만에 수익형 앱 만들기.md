## 1. 커서 AI를 활용한 수익형 앱 및 웹사이트 구축 개요 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/3uSN3hHTSI8/0.jpg)
1. **프로그램 및 웹사이트 완성 결과 제시** 
  1. 코딩 한 줄 없이 단 한 시간 만에 이미지 확장자 변환 프로그램과 서비스 소개 웹사이트를 만들었음을 강조한다. 
  2. 이 프로그램은 이미지 파일을 드래그하여 원하는 확장자로 변환할 수 있으며, 여러 개의 파일 변환도 지원한다. 
  3. 서비스 소개 웹사이트 역시 커서 AI를 활용하여 제작되었다. 

2. **수익형 웹사이트****의 잠재력 소개** 
  1. 이미지 확장자 변환 사이트인 '컨버트 아오(Convert Ao)'를 예시로 든다. 
  2. 통계 사이트 확인 결과, 해당 사이트의 하루 방문자는 <mark>170만 명</mark>에 달하며, 광고를 달 경우 하루에 <mark>천만 원 이상</mark>의 수익이 가능할 것으로 추정된다. 
  3. '컨버트 아오'는 변환 기능을 세부적으로 제공하고 구글 최상위권 순위를 차지하며 인지도를 쌓은 성공적인 사례이다. 
  4. 의외로 그리 어렵지 않지만, 소소한 불편함을 해소하는 사이트들이 인기가 많으며, 발표자 역시 확장자 변환 프로그램 설치가 귀찮아 해당 사이트를 이용하고 있다. 
  5. 이러한 수익형 사이트를 커서 AI로 만든 과정을 보여줄 것임을 예고한다. 


## 2. 커서 AI 환경 설정 및 컴포저 에이전트 활용 준비 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/3uSN3hHTSI8/66.jpg)
1. **커서 AI**** 도입 배경 및 요금제 정보** 
  1. 발표자는 기존에 챗GPT를 활용하여 빠른 개발을 해왔으나, 요즘 개발자들 사이에서 주목받는 **커서 AI**를 직접 사용해 보았다. 
  2. 커서 AI는 처음 설치 시 14일 동안 유료인 **프로 요금제**를 무료로 사용할 수 있다. 
  3. 무료 요금제도 존재하지만 기능이 떨어지고 사용량이 제한된다. 
  4. 커서 AI는 일반 에디터와 유사하지만 AI 기능이 추가된 것이 특징이다. 

2. **커서 AI****의 핵심 기능: 컴포저 에이전트** 
  1. 커서 에디터가 주목받는 이유는 **컴포저 에이전트** 기능 덕분이다. 
  2. 이 기능을 사용하면 한 번의 지시로 여러 파일이 자동으로 생성된다. 

3. **실시간 작업을 위한 서버 환경 구축** 
  1. 커서 AI가 웹사이트에 파일을 올려 처리하는 과정을 실시간으로 확인하기 위해 사전에 개인 아마존 서버에 새로운 웹사이트를 세팅해 두었다. 
  2. 서버에 SSH 접속하기 위해 **리모트 SSH**** 확장 도구**를 설치했다. 

4. **커서 AI**** 실행 및 모델 지정** 
  1. 커서 에디터에서 AI를 실행하려면 Ctrl+I를 누르면 된다. 
  2. 실행 시 **챗, 컴포저, 버그 파인더** 세 가지 탭이 나타나며, 특히 **컴포저 에이전트** 기능이 핵심이다. 
  3. 컴포저 에이전트에서는 만들고 싶은 것을 설명만 하면 바로 제작해 준다. 
  4. 커서 AI 실행 시 사용할 모델을 지정할 수 있으며, 이는 도구 자체보다 모델의 성능에 따라 코드 품질이 결정되므로 매우 중요하다. 
  5. 발표자는 여러 모델 중 **o3 mini**를 선택했다. 
  6. 세팅 아이콘을 클릭하면 클라우드, DeepSpeed 등 더 많은 모델을 볼 수 있다. 
  7. API 키를 넣으면 무료 버전을 사용하면서 사용한 양만큼만 비용을 지불하는 방식도 가능하며, 이는 부업이나 취미 개발자에게 더 경제적일 수 있다. 
  8. 발표자 역시 트라이얼 기간이 끝나면 이 방식으로 사용할 예정이다. 

5. **AI 지침 설정 (Rules for AI)** 
  1. 일반 AI 사용 시 지침을 적어두면 챗 진행 시 항상 그 지침을 지켜주는 것처럼, 커서에도 **"Rules for AI"** 메뉴가 있다. 
  2. 이 메뉴에 서버 환경 정보를 넣어두었다. 
  1. PHP 버전과 MySQL 버전 
  2. 설치된 PHP 모듈 
  3. MySQL 접속 계정 
  3. 환경 정보를 설정해 두면 AI가 버전에 맞게 파일을 생성하여 오류 없이 한 번에 파일을 만들 수 있다. 
  4. 버전이 틀릴 경우 코드 자체는 문제없어도 미지원으로 에러가 발생할 수 있으므로, 이러한 환경 세팅은 필수로 보인다. 
  5. 왼편 폴더 탐색기에는 아마존 서버의 폴더와 파일들이 작업 환경으로 표시된다. 


## 3. 웹사이트 및 이미지 변환 기능 개발 과정 (30분~1시간) 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/3uSN3hHTSI8/213.jpg)
1. **무료 테마**** 설치 및 웹사이트 제작 지시** 
  1. 컴포저 탭으로 이동하여 이미지 드래그 앤드 드롭 변환 기능과 웹사이트 제작을 요청할 계획을 세운다. 
  2. 웹사이트를 제대로 만들기 위해 구글에서 찾은 **무료 테마**를 서버에 설치했다. 
  1. 이 테마는 모바일에서도 최적화된 반응형이라 만족스러웠다. 
  3. 웹 브라우저로 확인 시 무료 테마의 웹사이트가 보였다. 
  4. 컴포저에 만들 내용을 설명하고 웹사이트 제작을 요청했으나, AI가 무료 테마를 변경하지 않고 새로운 파일을 만들었다. 
  5. 이에 "무료 테마를 편집해서 해 달라"고 구체적으로 지시하자 정상적으로 진행되었다. 
  6. 컴포저 실행 시 **에이전트 모드**를 누른 상태로 두어야 AI가 혼자서 작업을 진행한다. 
  7. 약 5분 정도 기다리자 AI가 자동으로 파일을 편집해 주었다. 
  8. AI가 최종 결과만 보여주는 것이 아니라, 바뀌는 내용을 보여주고 사용자가 승인하는 과정이 있어 현명한 방식이라고 평가된다. 
  9. 완성된 웹사이트는 설치한 테마 느낌으로 나왔으며, 반응형으로 모바일, 태블릿 등에서 예쁘게 표시되었다. 
  10. 시작한 지 <mark>30분 만에</mark> 웹사이트가 완성되었다는 사실에 놀라움을 표하며, 개발 분야에서 이처럼 빠른 변화는 없었던 것 같다고 언급한다. 

2. **이미지 변환**** 기능 추가 및 오류 수정** 
  1. 이미지 변환 기능은 별도로 만들어 달라고 요청했고, 해당 기능만 새 페이지에서 진행되었다. 
  2. **첫 번째 오류**: 확인 결과 상단 메뉴가 통째로 사라져 있었다. 
  3. **해결**: 곧바로 "메뉴도 넣어 줘"라고 지시하니 에이전트가 해결해 주었다. 
  4. 이미지 변환 오류 역시 같은 방식으로 간단히 수정했다. 
  5. **두 번째 오류**: 한 개의 이미지는 변환이 잘되지만, 두 개 이상은 변환되지 않았다. 
  6. **해결**: 다시 구체적으로 "두 개 이상일 때도 잘되도록" 지시하니 멀티 파일 변환도 정상적으로 진행되었다. 
  7. 완성된 웹사이트는 약 <mark>한 시간 만에</mark> 완성되었다. 

3. **개발 요약 및 향후 확장 가능성** 
  1. **단시간 완성의 세 가지 요소**: 커서 에디터, 무료 테마, 웹 서버에서의 실시간 작업 환경을 결합하여 단시간에 완성했다. 
  2. 현재 결과에 매우 만족하며, 여기에 이미지 용량 조절, 사이즈 조절, 문서 이미지 변환과 같은 여러 기능을 더 추가하면 완벽한 수익형 웹사이트가 될 것으로 예상된다. 
  3. 시도를 권유하며 마무리한다. 


## 4. 커서 AI 사용 후기 및 실무 팁 


### 4.1. 커서 AI의 기능별 특징 및 활용법 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/3uSN3hHTSI8/346.jpg)
1. **챗 기능 (조언 얻기)** 
  1. 컴포저 옆에 있는 **챗** 기능은 AI에게 무엇이든 물어볼 수 있는 기능이다. 
  2. **코드 베이스**를 선택하여 질문하면 AI가 내 파일들 전체를 살펴보고 그에 맞는 답변을 해 준다. 
  3. 기존 코드에서 작업할 때 유용하며, 챗 기능은 AI에게 조언을 얻는 용도로 사용된다. 

2. **컴포저 탭의 모드 구분** 
  1. 컴포저 탭에는 **노멀 모드**와 **에이전트 모드**가 있다. 
  2. **에이전트 모드**는 자동을 의미하며, AI가 알아서 파일들을 만들고 진행한다. 
  3. **노멀 모드**는 수동을 의미하며, 사용자의 지시를 기다린다. 
  4. 신중한 작업을 할 때는 노멀 모드가 더 나을 수 있다. 


### 4.2. AI 사용 시 아쉬웠던 점과 대처법 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/3uSN3hHTSI8/390.jpg)
1. **AI의 멍청한 행동과 거짓말** 
  1. 커서 편집기 역시 일반 AI처럼 멍청한 느낌이 있었다. 
  2. **예시 1**: 실제로 존재하지 않는 이미지를 불러오려고 시도했다. 
  3. **예시 2**: "적용 완료"라고 하면서 아무것도 하지 않는 경우도 있었다. 
  4. **예시 3**: 이미 적용했다고 거짓말하기도 했다. 
  5. **대처법**: 이런 경우 파일을 끌어와 올려놓고 얘기하면 그제서야 파악하고 수정해 주었다. 
  6. **예시 4**: 존재하지 않는 배경 때문에 웹페이지가 제대로 보이지 않았는데, 이를 설명해도 AI가 이해하지 못했다. 
  7. **해결**: 결국 개발자 도구에서 이미지 로드 에러를 발견하고 나서야 해결할 수 있었다. 

2. **진행 내역 누적 시 성능 저하** 
  1. 컴포저 진행 내역이 많이 쌓이면 AI가 헷갈려 하거나 완성도가 떨어지는 코드를 생성했다. 
  2. 데이터가 많아지면 헷갈려 하는 느낌이 들었다. 
  3. **해결**: 이럴 경우 커서를 재시작하면 다시 잘 작동했다. 

3. **리모트 SSH**** 설정의 어려움** 
  1. 영상에서는 리모트 SSH 확장을 사용했는데, 아마존 서버는 키 파일을 이용해서 SSH 연결을 하므로 이에 맞게 세팅하는 데 어려움이 있었다. 


### 4.3. AI 개발의 의미와 속도 혁신 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/3uSN3hHTSI8/459.jpg)
1. **개발자 지식의 도움** 
  1. 발표자가 웹사이트를 빨리 만든 것은 개발자로서의 기존 지식들이 도움이 된 것이 사실이다. 

2. **비개발자의 개발 가능성** 
  1. 코딩을 잘 모르는 사람들도 개발이 가능하다고 믿는데, 이는 코딩을 공부하고 AI가 막히는 부분을 해결하다 보면 분명 가능해 보이기 때문이다. 
  2. AI를 활용하면 이전에 비해 획기적으로 개발 기간을 단축할 수 있다. 
  3. 개발 독학도 가능해 보이며, 모르는 것은 AI에게 1대1 과외를 받는다고 생각하면 된다. 

3. **개발 속도****의 극대화 사례** 
  1. 이전 영상에서 3주 만에 파이썬으로 오토 블로그 프로그램을 만들었다고 소개했으나, 이번에는 <mark>일주일 만에</mark> 그것을 반응형 웹사이트로 업그레이드했다. 
  2. 이러한 속도감은 어마어마하다. 
  3. 이 작업은 **ChatGPT 3.5 모델**과 **GPT-3 미니 모델**을 사용하여 단시간에 진행되었다. 


## 5. AI 기술 발전 전망 및 마무리 
![image]()
1. **AI 성능**** 향상 속도** 
  1. 최근 GPT-4의 성능이 이슈가 되고 있으며, 이제 1년 주기가 아닌 몇 개월 사이로 AI 성능이 대폭 향상되고 있다. 
  2. 이러한 속도에 살짝 무섭다는 느낌을 받는다. 

2. **마무리** 
  1. 앞으로도 개발 분야에 대한 창의적인 영상을 계속 올릴 예정이다. 
  2. 좋아요와 구독을 부탁하며, 소스 놀이터의 다음 영상이 계속 이어질 것임을 알린다.
