## 📌 클로드 코드(Claude Code)는 어떤 서비스이며, 기존 AI 코딩 도구와 비교했을 때 어떤 장점이 있나요?
 클로드 코드는 API 키로만 사용 가능했던 클로드의 코딩 특화 버전으로, <mark>MCP(Multi-modal Control Plane) 기능과 최적화된 토큰 사용량</mark>을 통해 커서나 Replit 같은 기존 ai 에이전트보다 뛰어난 성능과 효율성을 제공합니다. 


## 💡 클로드 코드의 주요 특징과 장점은 무엇인가요?

- **압도적인 성능**: 그림판 만들기 예시에서 한 시간 만에 레이어 기능까지 구현하며 오류가 거의 없었음.
- **효율적인 토큰 사용**: 토큰을 최적화하여 클로드 데스크탑보다 작업을 오래 지속할 수 있음.
- **MCP 지원**: 플레이라이트 MCP로 브라우저를 직접 띄워 디버깅 및 실시간 오류 확인 가능.
- **긴 파일 처리**: 파일 길이 제한 문제 없이 긴 코드도 처리 가능.

클로드 코드는 기존 AI 코딩 도구의 한계를 뛰어넘는 **압도적인 성능과 효율성**을 제공합니다. 특히, **MCP(Multi-Context Programming) 기능**을 활용하여 브라우저를 직접 띄우고 실시간 디버깅까지 수행하는 능력은 코딩 오류를 획기적으로 줄여주며, 복잡한 포토샵 레이어 시스템을 단 한 시간 만에 구현해내는 놀라운 작업 속도를 경험하게 합니다. 토큰 소모를 최적화하고 길이 제한 문제를 해결한 이 도구를 통해, 개발자는 더 이상 잦은 중단 없이 **프로젝트의 본질적인 개발**에만 집중할 수 있는 실질적인 도움을 얻게 될 것입니다.
## 1. 클로드 코드(Claude Code)의 등장과 압도적인 성능 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UpSlgSoBcMo/0.jpg)
1. **클로드 코드****의 출시 및 접근성 확대** 
  1. 클로드 코드는 기존에 API 키로만 사용 가능했으나, 최근 <mark>100달러 플랜 유저</mark>와 <mark>20달러 프로 유저</mark>에게도 사용이 허용되었다. 

2. **경쟁 제품 대비 우위** 
  1. 며칠 사용해 본 결과 매우 만족스러우며, 비싼 AI 에이전트인 마누스(Manus)나 레플릭(Replic)과 비교해도 전혀 밀리지 않거나 오히려 더 좋은 성능을 보여준다. 

3. **그림판 제작 사례를 통한 성능 입증** 
  1. **요청 내용**: 포토샵 기능 10가지(선 긋기, 사각형, 원 그리기, 스프레이, 텍스트 입력)를 포함한 그림판 제작을 요청했다. 
  2. **구현된 핵심 기능**: 포토샵의 핵심 기능인 <mark>레이어 기능</mark>까지 구현되었으며, 레이어 순서 바꾸기 및 투명도 조절까지 가능하여 웹에서 포토샵을 사용하는 느낌을 주었다. 
  3. **놀라운 작업 속도**: 이미지 저장 및 프로젝트 자체 저장이 가능한 이 기능을 <mark>단 한 시간 만에</mark> 완성했다. 
  4. **오류 최소화 입증**: 일반적으로 이 정도 기능을 구현하는 데 오랜 시간이 걸리는 점을 고려할 때, 한 시간 만에 끝났다는 것은 작업 중 오류가 거의 없었음을 의미한다. 

4. **사용량 제한 문제 해결 및 MCP 기능 활용** 
  1. **사용량 제한 해소**: 20달러 플랜 사용 중에도 단 한 번도 작업이 멈추지 않았으며, 기존 클로드에서 과학 작업을 할 경우 5시간을 쉬어야 했던 한계가 클로드 코드에서는 발생하지 않았다. 
  2. **MCP(Multi-Context Programming) 기능**: MCP 기능을 사용하여 플레이라이트(Playwright)로 <mark>브라우저를 직접 띄우고</mark> 스스로 확인하며 디버깅까지 수행할 수 있어 코딩 작업에 매우 유용하다. 


## 2. 클로드 코드의 효율성 및 설치 과정 
![image]()
1. **토큰 소모**** 최적화의 장점** 
  1. 클로드 데스크탑은 토큰 소모가 많아 제한에 빨리 도달하는 단점이 있다. 
  2. 반면, 클로드 코드는 토큰을 최적화하여 작업을 더 오래 지속할 수 있게 한다. 
  3. 초기 사용 시에는 다소 불편할 수 있으나, 익숙해지면 완전히 프로페셔널한 느낌을 준다. 

2. **클로드 코드**** 설치 준비 (WSL 설치)** 
  1. 클로드 코드는 리눅스 환경에서 구동되므로, 윈도우 사용자는 가상 리눅스를 설정하는 <mark>WSL(Windows Subsystem for Linux)</mark>부터 설치해야 한다. 
  2. 명령 프롬프트를 열고 WSL install을 입력하여 설치를 진행하며, 설치 도중에 자주 사용하게 될 아이디와 비밀번호를 설정하고 기억해야 한다. 
  3. 설치가 끝나면 재부팅 후 다시 명령 프롬프트에서 WSL을 입력하여 가상 리눅스 환경에 진입한다. 

3. **클로드 코드**** 설치 및 로그인** 
  1. 가상 리눅스 환경에서 세 가지 명령을 차례대로 입력하면 클로드 코드 설치가 완료된다. 
  2. WSL에서 클로드를 입력하면 "Welcome to Claude Code" 메시지와 함께 실행된다. 
  3. **로그인 방법**: 프로 플랜 사용자는 1번을 선택하며, 유료 플랜 고객은 추가 요금이 발생하지 않는다. 
  4. 긴 URL이 뜨면 복사하여 브라우저에 붙여 넣고 승인 버튼을 클릭한다. 
  5. 화면에 나타난 코드를 복사하여 클로드 코드에 붙여 넣으면 로그인이 성공한다. 

4. **작업 환경 확인 (****MNTC****와 ****CWD****)** 
  1. **MNTC**: 화면에 보이는 MNTC는 윈도우의 C드라이브를 가상 리눅스 환경에 마운트(Mount)한 것이다. 
  2. **CWD**** (Current Working Directory)**: 중간에 보이는 CWD는 현재 프로젝트 폴더를 의미하며, 이는 클로드 데스크탑의 프로젝트와 동일한 개념이다. 
  3. 클로드 코드는 앞으로 이 CWD 폴더 안에서만 파일을 생성하게 된다. 
  4. CWD는 <mark>클로드 명령어를 입력한 순간의 경로</mark>로 설정되므로, CD 명령어로 프로젝트 폴더로 이동한 후 클로드를 입력해야 한다. 


## 3. 프로젝트 초기화 및 환경 설정 
![image]()
1. **초기화의 중요성 (슬래시 이닛)** 
  1. 클로드 코드를 실행한 후 바로 작업을 시작하는 것이 아니라, 먼저 /이닛 명령어로 초기화를 수행해야 한다. 
  2. 이 초기화 작업을 통해 <mark>클로드 마크다운 파일</mark>이 생성되며, 클로드 코드는 항상 이 파일을 참고하여 작업을 진행한다. 
  3. 이 파일에 프로젝트 지침을 넣으면 클로드가 더 효율적이고 똑똑하게 작업을 수행할 수 있다. 

2. **기존 프로젝트 요약 및 초기화** 
  1. 이미 클로드 데스크탑에서 개발해 온 소스 파일들이 있는 경우, 클로드 코드로 이어서 작업을 진행할 수 있다. 
  2. /이닛 전에 영어로 summarize this project라고 입력해야 하며, 클로드 코드가 이 영어 표현에 훈련되어 있어 더 잘 인식한다. 
  3. 클로드는 전체 파일을 분석하여 해당 프로젝트가 '유튜브 쇼츠 생성기'임을 파악한다. 
  4. 이후 /이닛을 실행하면 방금 분석한 내용과 코드의 특징을 정리하여 클로드 마크다운 파일에 기록한다. 

3. **클로드 마크다운 파일 편집 및 최적화** 
  1. 생성된 클로드 마크다운 파일은 에디터에서 확인 및 편집할 수 있다. 
  2. 클로드 데스크탑에서 사용하던 프로젝트 지침을 복사하여 넣되, 클로드 코드는 자체 에디터가 있으므로 <mark>에디터 관련 내용은 모두 삭제</mark>한다. 
  3. 클로드 데스크탑에서 에디터 관련 오류가 많았던 고민을 해소할 수 있다. 
  4. C드라이브 경로는 마운트 아래의 C 경로로 깔끔하게 수정한다. 
  5. 이 과정을 거치면 모든 준비가 완료되어 작업을 시작할 수 있다. 


## 4. 작업 중 기능 및 효율성 (토큰 관리, 길이 제한, MCP 설정) 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UpSlgSoBcMo/260.jpg)
1. **작업 중 ****권한 요청**** 처리** 
  1. 작업을 진행하다 보면 특정 작업을 계속 진행해도 되는지 묻는 권한 요청 메시지가 나타난다. 
  2. **옵션**: 1번은 한 번만 허락, 2번은 앞으로 같은 작업은 모두 허락, 3번은 작업을 허락하지 않는 것이다. 
  3. 허락하면 작업이 바로 이어서 진행된다. 

2. **잔여 토큰 관리 및 ****컨텍스트 압축** 
  1. 화면 오른쪽 하단에 남은 잔여 토큰(예: 6%)이 표시된다. 
  2. 잔여 토큰이 0%가 되면 클로드 코드는 자동으로 컨텍스트 압축을 진행한다. 
  3. 압축을 통해 여유 공간이 확보되어 작업을 계속 이어나갈 수 있다. 
  4. 사용자는 잔여 토큰을 확인하고 컨텍스트가 다 차기 전에 작업을 정리하는 등 필요한 대처를 할 수 있다. 

3. **파일 길이 제한**** 문제 해결** 
  1. 클로드 코드는 파일 길이가 너무 길어서 중간에 끊기는 문제가 없다는 것이 큰 장점이다. 
  2. 클로드 데스크탑에서 자주 발생했던 파일 길이 제한 문제가 해소되어 매우 편리하다. 

4. **MCP 설치 및 설정 변경** 
  1. **MCP 가져오기**: WSL 상태에서 특정 명령어를 입력하면 클로드 데스크탑에 있는 MCP들을 한 번에 가져올 수 있으며, 가져올 MCP를 선택 후 엔터를 누른다. 
  2. **설치 실패 시 대처**: 설치가 실패한 MCP가 있을 경우 설정 파일을 확인해야 한다. 
  3. **설정 파일 경로**: 파일 탐색기에서 리눅스 폴더를 열고, 홈 폴더 아래 사용자명 폴더로 들어가면 클로드 설정 파일이 있다. 
  4. **경로 수정**: 편집기로 설정 파일을 열면 MCP 서버 항목이 보이지만, 경로가 윈도우 기준으로 되어 있어 수정이 필요하다. 
  1. C 드라이브 경로는 mount/C로 변경한다. 
  2. 백슬래시 두 개는 슬래시 하나로 변경한다. 
  5. **MCP 확인**: 클로드 코드를 실행하고 /cp를 입력하면 설치된 MCP들을 모두 확인할 수 있다. 


## 5. 작업 모드 활용 및 디버깅 혁신 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/UpSlgSoBcMo/364.jpg)
1. **작업 모드 (****플랫 모드****와 ****자동 실행 모드****)** 
  1. 클로드 코드에는 플랫 모드와 자동 실행 모드라는 작업 모드가 있다. 
  2. 이는 슈림프 테스크(Shrimp Task)와 유사한 방식이지만, 클로드 코드의 성능이 좋다면 토큰 소모가 심한 슈림프가 필요 없을 수도 있다. 
  3. 지속적인 사용을 통해 성능을 확인할 예정이다. 

2. **프로젝트 플랜 파일**** 활용 및 모드 전환** 
  1. **상황 기록**: 클로드 코드는 어디까지 작업했는지 물어보면 상세히 알려준다. 
  2. **플랜 파일 저장**: 슈림프 대신 프로젝트 플랜 파일을 활용하여 필요할 때마다 현재 상황을 저장하게 했으며, 새 창을 열었을 때도 현재 상황을 바로 파악할 수 있다. 
  3. **플랫 모드**** (작업 계획)**: Shift+Tab을 눌러 플랫 모드로 전환할 수 있으며, 이 모드에서는 작업 계획을 짜준다. 
  1. 약 1분 정도 조사 후 예상 작업 시간과 복잡도까지 포함된 계획이 깔끔하게 정리되어 나온다. 
  4. **싱킹 모드 (심층 조사)**: 더 구체적인 계획이 필요할 때 "싱크하드(Sync Hard)"를 입력하면 싱킹 모드로 바뀌면서 깊게 조사하기 시작한다. 
  1. 이 모드에서는 더 자세히 분석하기 위해 시간을 두 배나 들이며, 그 결과물도 더 상세하다. 
  5. **오류 감소 효과**: 이러한 플랜 과정을 거치면 실제 코딩 시 오류가 현저히 줄어든다. 
  6. 클로드 데스크탑에서 자주 발생하던 길이 제한 오류가 없다는 점이 큰 장점이다. 

3. **MCP를 활용한 실시간 ****디버깅**** (그림판 제작 사례)** 
  1. **요청**: 포토샵을 조사하여 비슷하게 만들되, 기능은 10개만 넣어 달라고 요청했다. 
  2. **디버깅**** 과정**: 플레이라이트 MCP를 이용하여 직접 디버깅하면서 진행하도록 요청했으며, 브라우저가 잘 뜨는 것이 확인된다. 
  3. 클로드 코드는 스스로 상태를 점검하고 디버깅을 직접 수행한다. 
  4. **실시간 오류 확인**: 브라우저 콘솔 로그를 통해 실시간 오류 확인이 가능하며, 콘솔에서 자바스크립트를 즉석에서 작성하며 실시간 테스트도 진행한다. 
  5. 코드가 어떻게 바뀔지 바로바로 보여주며, 중간중간 프로젝트 플랜 파일에 현재 작업 상황을 기록하게 했다. 

4. **레이어 시스템**** 구현 및 최종 결과** 
  1. **레이어 시스템**** 구현**: 처음에 포토샵처럼 해 달라고 요청했을 때, 복잡한 레이어 시스템까지 만들어 낼 줄은 예상하지 못했다. 
  2. 구현된 레이어 시스템은 작동이 매우 깔끔하다. 
  3. **최종 완료**: 약 한 시간 만에 모든 작업이 완료되었으며, 이는 오류가 거의 없었음을 다시 한번 입증한다. 
  4. 토큰 소모로 인해 작업이 멈춘 적도 전혀 없었으며, 이는 매우 훌륭한 경험으로 평가된다.
