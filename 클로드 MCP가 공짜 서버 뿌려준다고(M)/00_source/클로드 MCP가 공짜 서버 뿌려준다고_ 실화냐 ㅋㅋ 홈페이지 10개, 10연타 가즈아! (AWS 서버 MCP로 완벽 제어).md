## 📌 클로드 MCP를 활용하여 AWS 서버에 무료로 웹사이트를 구축하는 방법은?
 클로드 mcp를 통해 aws<mark> 프리티어 서버에 </mark>워드프레스<mark> 웹사이트를 20분 만에 무료로 설치하고 도메인 및 </mark>ssl<mark> 보안 연결까지 설정</mark>할 수 있습니다. 


## 💡 이 과정에서 필요한 주요 단계는?

- Node.js 및 SSH MCP 설치
- AWS 서버(EC2) 생성 및 키페어, 보안 그룹 설정
- 클로드 데스크탑에 AWS 서버 정보 연동
- 클로드와 대화하며 워드프레스 설치, 도메인 연결, HTTPS 적용

AI 시대에 **웹 서버**** 구축과 관리**는 더 이상 전문가의 영역이 아닙니다. 이 콘텐츠는 클로드 mcp를 활용하여 **aws**** 프리티어 서버**에 워드프레스를 설치하고, 도메인 연결 및 무료 ssl 보안 인증까지 **단 20분 만에** 완료하는 전 과정을 상세히 보여줍니다. 코딩 지식 없이도 AI와의 대화만으로 **완전 ****자동화****된 웹사이트 세팅**을 경험하고, 서버 비용을 획기적으로 절감하는 실질적인 방법을 배울 수 있습니다.
## 1. AI (클로드 MCP)를 활용한 웹 서버 자동 구축 및 결과 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/0.jpg)
1. **AI를 통한 ****웹 서버 세팅****의 용이성 강조** 
  1. MCP(Claude MCP)를 사용하면 코딩이 완전 자동화되며, 웹 서버 세팅은 코딩보다 훨씬 쉬워 AI에게는 식은 죽 먹기이다. 
  2. 직접 시도한 결과, 워드프레스 자동 설치, 도메인 연결, HTTPS 보안 인증서 설치까지 **대성공**하였다. 

2. **자동화****된 ****웹사이트 구축**** 결과 및 소요 시간/비용** 
  1. **구축 결과**: 워드프레스가 자동으로 설치되었고, 도메인 연결 및 HTTPS 보안 인증서까지 설치된 웹사이트가 생성되었다. 
  2. **디자인 및 꾸미기**: 초기 디폴트 페이지는 허접해 보일 수 있으나, 테마 설치를 통해 쉽게 예쁘게 꾸밀 수 있으며 코딩 지식이 필요 없다. 
  3. **소요 시간**: 이 모든 과정은 고작 **20분** 만에 완료되었다. 
  4. **비용 절감**:
  1. 서버 비용은 **영원**이며, AWS 프리티어(처음 사용자 1년 무료)를 사용하여 세팅했기 때문에 추가 비용이 전혀 발생하지 않았다. 
  2. 도메인은 기존에 있던 것을 사용했으며, HTTPS 보안 인증서 역시 무료로 설치되었다. 
  5. **직접 세팅의 장점**: 비용 절약과 함께 웹사이트를 마음대로 꾸밀 수 있으며, MCP를 사용하면 누구나 쉽게 할 수 있다. 


## 2. 클로드 MCP 설치 및 AWS 서버 생성 준비 


### 2.1. SSH MCP 설치 및 클로드 데스크탑 설정 준비 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/60.jpg)
1. **Node.js 설치**: Node.js 홈페이지에 접속하여 다운로드 및 설치를 완료해야 한다. 
2. **SSH**** MCP 설치**:
  1. 설치 후 명령 프롬프트를 열고, 제공된 명령어를 한 줄씩 복사 붙여넣기하여 실행하면 SSH MCP가 설치된다. 
3. **클로드 데스크탑 설정 파일 준비**:
  1. 클로드 데스크탑의 설정 파일을 메모장으로 열고 특정 내용을 추가해야 한다. 
  2. 이 설정에 필요한 **호스트(Host)**와 **키(Key)**는 아마존 서버를 생성해야 얻을 수 있다. 

4. **AWS**** 서버 생성 단계 예고**: 지금부터 아마존 서버를 만드는 방법을 안내한다. 


### 2.2. AWS EC2 인스턴스 생성 및 키페어 확보 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/95.jpg)
1. **AWS**** 접속 및 로그인**: AWS 콘솔을 검색하여 아마존 AWS에 접속하고 회원 가입 및 로그인을 진행한다. 
2. **EC2 인스턴스 시작**:
  1. EC2 대시보드로 이동하여 오렌지색 **인스턴스 시작** 버튼을 누른다. 
  2. 서버 종류는 **우분투 서버**를 선택하며, 버전 선택 시 **프리티어**라고 쓰인 것을 선택해야 1년 무료 혜택을 받을 수 있다. 
3. **키페어**** 생성 (매우 중요)**:
  1. **새 ****키페어**** 생성**을 누르고 키페어 이름을 적는다. 
  2. PM이 선택된 상태로 생성을 누르면 파일이 다운로드된다. 
  3. 이 키페어 파일은 SSH 로그인 시 비밀번호 대신 사용되므로 **절대 잃어버리면 안 된다**. 
  4. 방금 생성한 키를 선택한다. 


### 2.3. 보안 그룹 설정 및 인스턴스 시작 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/142.jpg)
1. **보안 그룹**** 설정**:
  1. 보안 그룹 생성을 누른 후, **SSH**** 트래픽 허용** 부분에서 **내 IP**를 선택한다. 
  2. 이는 내 IP에서만 SSH 로그인이 가능하게 하여 안전성을 높이는 조치이며, SSH를 모든 IP에 열면 해킹 위험이 상당히 올라가므로 주의해야 한다. 
2. **인스턴스 시작**: 나머지 설정은 건드리지 않고 기본 설정으로 유지한 후 인스턴스 시작을 누른다. 
3. **퍼블릭 IP**** 확보**:
  1. 서버가 만들어지면 인스턴스 메뉴로 이동하여 실행 중인 서버를 클릭한다. 
  2. 세부 정보에서 **퍼블릭 IP**** 4 주소**를 찾아 기억해야 하며, 이 IP로 서버에 접속하게 된다. 


## 3. 클로드 MCP 설정 완료 및 AI를 통한 서버 자동 세팅 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/185.jpg)
1. **클로드 데스크탑 설정 업데이트**: 
  1. 클로드 데스크탑의 파일 메뉴에서 설정으로 들어가 개발자 탭을 클릭하고 설정 파일을 메모장으로 연다. 
  2. MCP Servers 항목 맨 마지막 부분에 새로운 내용을 추가한다. 
  3. **호스트(Host)**에는 AWS에서 확보한 퍼블릭 IP4 주소를 넣고, **포트(Port)**는 22, **유저(User)**는 우분투로 입력한다. 
  4. **키(Key)**에는 앞서 저장해 둔 키페어 위치를 적어주는데, 이때 **백슬래시를 두 번** 넣어야 한다. 
2. **클로드 재시작 및 ****SSH**** MCP 확인**: 
  1. 설정을 완료했으면 클로드 아이콘을 클릭하여 종료 후 다시 실행해야 한다. 
  2. 에러가 없으면 성공이며, 툴 메뉴를 눌러 **SSH**** MCP**가 보이면 성공적으로 설치된 것이다. 
3. **클로드와의 대화를 통한 서버 세팅 (전자동)**: 
  1. **접속 및 계정 생성**: 클로드에게 SSH로 서버에 접속해 달라고 요청한 후, 최고 관리자인 루트로 변경하여 웹사이트를 만들 **마이 홈페이지** 계정을 만들어 달라고 요청한다. 
  2. **워드프레스**** 설치 요청**: 클로드에게 무료 솔루션인 워드프레스를 설치해 달라고 부탁하며, 필요한 것을 알아서 다 설치해 달라고 요청한다. 
  3. **자동 설치 과정**: 클로드가 알아서 척척 진행하며, 가끔 에러가 나도 대화하다 보면 스스로 해결한다. 
  1. 사용자 생성 후, 웹 서버인 **아파치**, 데이터베이스인 **MySQL**, 그리고 **PHP** 등 필요한 프로그램들을 차례차례 설치한다. 
  2. DB 설정까지 알아서 진행하며, 워드프레스를 스스로 다운로드하고 설치까지 진행한다. 
  4. **설치 완료**: 설치가 완료되면 클로드가 IP 주소를 알려준다. 


## 4. AWS 추가 설정 및 도메인/SSL 연결을 통한 보안 강화 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/289.jpg)
1. **AWS**** ****인바운드 규칙**** 설정 (80번 포트 개방)**: 
  1. AWS로 돌아가 인바운드 규칙에 **80번 포트**를 열어야 한다. 
  2. 80번 포트는 홈페이지 서비스를 제공하는 포트이므로, 이를 모든 사람에게 열어주기 위해 **0.0.0.0**으로 설정하고 규칙을 저장한다. 
  3. 브라우저에 IP 주소를 입력하면 워드프레스 설정 화면이 뜨는 것을 확인할 수 있다. 

2. **도메인 연결**** 설정**: 
  1. IP 주소로 접속하는 것은 아쉽고 느낌표 표시(보안 취약)도 거슬리므로 도메인을 연결하고 SSL을 설정하여 안전하게 만들 계획이다. 
  2. **도메인 설정 원리**: 도메인은 서버 IP를 향하게 하고, 서버는 도메인을 가리키게 하여 서로 양방향으로 연결한다. 
  3. **도메인 레코드 설정 (****단네임**** 예시)**:
  1. 도메인 구매 사이트(예: 단네임)에 로그인하여 DNS 레코드 설정 메뉴로 들어간다. 
  2. 레코드 서비스 추가를 누르고, 접두어로 AWSMCP를 입력한 후 IP는 홈페이지 IP를 적어주어 도메인이 서버 IP를 가리키게 한다. 
  3. 다른 사이트도 기본 원리는 동일하며, 메뉴 이름이 다를 경우 고객 센터에 문의하면 된다. 

3. **클로드를 통한 서버 도메인 인식 설정**: 
  1. 서버에서도 이 도메인을 인식하게 해야 하며, 이는 MCP로 간단히 해결할 수 있다. 
  2. 클로드 데스크톱을 열고 "사이트 도메인을 awmmcp.g.kr로 설정해 줘"라고 말하면 클로드가 바로 처리해 준다. 
  3. 웹 브라우저에 도메인을 입력하면 사이트가 잘 뜨는 것을 확인하여 성공한다. 

4. **무료 ****SSL**** 적용 및 ****HTTPS**** 설정**: 
  1. 도메인 연결 후에도 느낌표가 남아있다면 보안 연결(SSL)이 안 된 상태이다. 
  2. **SSL**** 적용**: 원래 SSL은 유료이지만 직접 설치하면 무료이므로, 클로드에게 "HTTPS 적용시켜 줘"라고 부탁하고, "http로 접속하면 자동으로 HTTPS로 바꿔 줘"라고 요청한다. 
  3. **AWS**** 443번 포트 수동 개방**: SSL 포트인 **443번 포트**는 수동으로 열어야 하므로, AWS로 돌아가 443번 포트를 모든 대상으로 설정하여 열어준다. 
  4. 중간에 에러가 발생하더라도 클로드가 스스로 해결해 준다. 
  5. 모든 설정이 끝난 후 사이트에 다시 접속하면 느낌표가 사라지고 안전해진 것을 확인할 수 있다. 


## 5. 워드프레스 최종 설정 및 AI 활용의 확장성 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/RgJ96bYMTmM/426.jpg)
1. **워드프레스**** 기본 설정 및 관리자 계정 생성**: 
  1. 워드프레스를 처음 열면 언어를 한국어로 선택하고 사이트 제목을 입력한다. 
  2. 사용자명과 비밀번호를 만들어야 하며, 이는 **관리자 아이디와 패스워드**이므로 안전한 곳에 메모해 두어야 한다. 
  3. 이메일 주소를 입력하고 설치 버튼을 누르면 설치가 성공하며, 관리자로 로그인하여 완벽하게 작동하는 것을 확인한다. 

2. **워드프레스****의 장점 및 AI 활용**: 
  1. 초기 홈페이지는 밋밋해 보이지만, 테마를 바꾸면 멋진 사이트가 완성된다. 
  2. 워드프레스는 코딩을 전혀 몰라도 버튼 클릭만으로 멋진 사이트를 만들 수 있다는 장점이 있다. 
  3. 처음에는 헷갈릴 수 있지만, AI에게 물어보면서 진행하면 금방 익숙해진다. 

3. **AI 시대의 서버 관리 변화 및 확장 가능성**: 
  1. 과거에는 해외 사이트를 구글링하고 영어 번역하며 힘들게 서버 세팅을 했지만, 이제 클로드가 뚝딱 해결해 준다. 
  2. **응용 가능성**: 이 방법을 응용하면 하나의 서버로 **수십 개의 웹사이트**를 운영할 수 있어, 한 달에 수십만 원의 비용을 엄청나게 절약할 수 있다. 
  3. 이러한 기술 활용을 통해 웹호스팅 사업도 가능할 것으로 추정된다. 
  4. 서버 관리자라는 직업 역시 예전에는 전문가만 할 수 있었으나, 이제는 누구나 쉽게 도전할 수 있도록 변화하고 있다.
