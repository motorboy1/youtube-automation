## 📌 편집바이브 코딩 '최강템' 4개를 활용해 '무료 레플릿' 시스템을 만드는 방법은?
 커서(Cursor)를 메인으로 클로드 코드(Claude Code)와 코덱스 cli(Codex CLI)를 함께 사용하며, 슈퍼베이스(Supabase)를 통해 데이터베이스 및 백엔드를 구축하여 레플릿과 유사한 코딩 환경을 만드는 것입니다. 


## 💡 이 시스템의 핵심 장점은 무엇인가?

- **편의성**: 커서 무료 버전으로 클로드 코드와 코덱스 CLI를 유연하게 연동하여 사용 가능.
- **효율성**: 슈퍼베이스가 데이터베이스 및 인증 등 복잡한 백엔드 작업을 자동 처리하여 개발 속도와 품질 향상.
- **성능**: 코덱스 CLI의 심층 분석 능력과 클로드 코드의 실행력을 결합하여 어려운 문제 해결 및 최적화된 작업 가능.

레플릿(Replit)의 고성능 코딩 환경을 **무료 AI 툴 조합**으로 완벽하게 재현하는 **'AI 코딩 최강 ****프레임워크****' 구축 노하우**를 공개합니다. 이 콘텐츠를 통해 커서(Cursor)를 중심으로 클로드 코드, 코덱스 cli, 그리고 supabase mcp를 결합하여, AI의 장점을 극대화하고 **토큰과 시간을 절약**하는 실전적인 멀티 AI 협업 시스템을 구축할 수 있습니다. 특히, 코덱스 cli의 심층 분석 능력과 클로드 코드의 실행력을 결합하는 **'투 트랙 전략'**과 Supabase를 활용해 데이터베이스 작업을 자동화하는 방법을 배워, 단 10시간 만에 실용적인 웹 애플리케이션을 완성하는 놀라운 생산성을 경험하게 될 것입니다.
## 1. 레플릿 시스템 분석 및 AI 코딩 프레임워크 구축 목표 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/0.jpg)
1. **레플릿****(Replit)의 성능 분석 및 단점 파악** 
  1. 레플릿은 코딩을 모르는 사람도 결과물이 좋아서 많이 사용하며, 클로드보다 결과가 더 좋을 때도 있다 
  2. 이는 레플릿이 단순히 말해도 코딩을 잘 진행하기 때문이지만, 그만큼 비싸다는 최대 단점이 있다 

2. **레플릿**** 고성능의 비밀: ****프레임워크** 
  1. 레플릿은 쉬운 기능에는 자체 AI 모델을 사용하고, 핵심 기능에는 클로드와 같은 유명 LLM(대규모 언어 모델)을 가져다 쓴다 
  2. 레플릿이 동일한 LLM을 사용하면서도 성능이 잘 나오는 이유는 **프레임워크** 덕분이며, 이는 잘 만들어진 여러 틀을 가져와 AI 특성에 맞게 시스템을 만든 것이다 
  3. 이 분석을 바탕으로 레플릿과 동일한 프레임워크를 구축하는 프로젝트가 시작되었다 

3. **AI 코딩 최강 ****프레임워크****의 핵심 전략 및 구성 요소** 
  1. 구축된 노하우를 모두 공개하며, **커서****(Cursor)**를 메인 툴로 사용하여 클로드 코드와 코덱스 CLI의 장점만 쏙쏙 뽑아낼 것이다 
  2. 커서 무료 버전만으로도 충분하며, 중요한 CLI 툴들을 아주 편하게 섞어 쓸 수 있어 비용을 절약할 수 있다 
  3. 특히, 커서에서 코덱스 CLI를 사용할 때 오늘 공개되는 내용을 모르면 코덱스 성능의 반도 못 쓰고 있을 가능성이 높다 


## 2. Supabase 설치 및 초기 환경 설정 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/92.jpg)
1. **Supabase 접속 및 프로젝트 설정** 
  1. 레플릿 시스템 구축에 가장 중요한 **Supabase**를 설치하기 위해 Supabase.com에 접속하여 로그인해야 한다 
  2. 프로젝트 설정 시 데이터베이스 패스워드와 리전을 선택하며, DB 패스워드와 프로젝트 URL을 반드시 잘 보관해야 한다 

2. **Supabase 액세스 토큰 생성** 
  1. 오른쪽 상단 프로필 아이콘을 클릭하고 'Account Preferences'를 선택한다 
  2. 'Access Tokens'를 누른 후 생성 버튼을 눌러 액세스 토큰을 발급받고, 이 토큰 역시 잘 보관해야 한다 

3. **클로드 코드****를 이용한 자동 설치** 
  1. 영상 댓글에 있는 자동 설치 지침을 복사하여 터미널에 붙여 넣고 클로드에게 설치를 부탁한다 
  2. 설치 과정에서 이전에 보관해 뒀던 프로젝트 URL과 액세스 토큰을 클로드에게 다시 알려주어야 했다 
  3. 클로드 코드가 어려움을 겪었으나 시간이 지나 설치에 성공했으며, 터미널에 /MCP를 입력하면 Supabase가 잘 보이는 것을 확인할 수 있다 

4. **클로드 설정 파일 수정 (편집 권한 부여)** 
  1. 설치가 완료되면 사용자 폴더에 .claude.json 파일이 추가된다 
  2. 이 파일에서 read_only 구문을 지워주는데, 이는 Supabase 데이터베이스에 삽입, 수정, 삭제 같은 **편집 권한**을 주기 위함이다 
  3. 개발 단계에서는 빠르고 편하게 작업하기 위해 이 권한을 부여한다 

5. **커서****(Cursor) 환경 설정** 
  1. 커서 무료 플랜을 사용하며, 클로드 코드와 함께 작업하기에 충분하다 
  2. 'Open Folder'를 눌러 작업할 폴더를 선택하고, Ctrl+Shift+P를 눌러 명령 팔레트를 띄운다 
  3. 명령 팔레트에 'terminal'을 입력하고 'Create New Terminal in Editor Area'를 선택하면 탭에 파워셀이 나타나 클로드 코드를 사용할 수 있다 
  4. **클로드 MD 파일 생성**: 클로드가 항상 기억하는 지침 파일인 claude.md 파일을 새로 생성한다 
  5. 이 파일에는 레플릿 시스템을 만들기 위한 UI 계층, 인증, 데이터베이스, 백엔드 등에 대한 모든 세부 사항을 상세하게 기술하여 레플릿 시스템과 최대한 비슷하게 준비한다 
  6. 이로써 기본 설정이 끝난다 


## 3. 커서 활용 꿀팁 및 가게부 앱 설계 단계 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/236.jpg)
1. **커서****-****클로드 코드**** 활용 꿀팁** 
  1. **드래그 영역 인식**: 커서에서 특정 부분을 마우스로 드래그하고 질문하면, 클로드가 드래그한 영역을 정확히 인식하여 답해준다 
  2. **부분 편집 요청**: 원하는 부분을 드래그한 다음 "여기만 편집해 줘"라고 요청하면, 딱 그 부분만 깔끔하게 수정되어 불필요한 수정을 막아준다 

2. **가게부 앱 개발 목표 설정** 
  1. 복잡하지 않고 필요한 기능만 넣은 가게부 앱을 만들기로 한다 
  2. 지출 내역은 영수증이나 텍스트로 입력할 것이라고 클로드에게 알려준다 
  3. 좋은 결과물을 위해 사용자에게는 사전 조사를 꼼꼼히 할 것을 추천한다 

3. **PRD 및 ****LLD 문서**** 생성 요청** 
  1. 앱 개발 전 설계부터 하기 위해 클로드에게 **PRD** (요구 사항 문서)와 **LLD** (상세 설계 문서)를 만들어 달라고 요청한다 
  2. 플랜 모드로 진행했으나, 문서 생성 과정이 빠져서 일단 계획을 승인하고 잠시 멈춘다 
  3. 추가 명령으로 "계획 문서, PRD, LLD 문서를 파일로 저장해 줘"라고 요청하여 기존 계획을 살리면서 원하는 문서를 얻고, 시간과 토큰을 절약한다 
  4. 잠시 후 세 가지 문서(PRD, LLD, 플랜 문서)가 완성되었다 

4. **문서 내용 및 클로드 지침 업데이트** 
  1. **PRD 문서**: 프로젝트의 목표와 핵심 기능이 정리되어 있다 
  2. **LLD 문서**: 개발자의 설계도와 같은 기술적인 내용이 담겨 있다 
  3. **플랜 문서**: 간단한 기술 스펙과 실행 단계들이 포함되어 있다 
  4. 클로드 마크다운 파일에는 "이 문서들을 바탕으로 개발하되 변경 사항도 계속 업데이트해라"라는 중요한 내용이 추가되었다 
  5. 이로써 중요한 첫 단계를 넘어섰다 


## 4. 1단계 진행 및 시스템 정비 (Git/메모리 관리) 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/356.jpg)
1. **1단계 진행 (모듈 설치)** 
  1. 첫 번째 단계를 진행해 달라고 명령하며 싱크하드(Sync Hard)라는 단어를 넣는다 
  2. 이 단어를 사용하면 토큰을 많이 써서 더 좋은 결과가 나오며, 클로드가 어떤 단계를 거칠지 자신의 생각을 보여주는 힌트도 제공되어 큰 도움이 된다 
  3. 클로드가 필요한 모듈들을 자동으로 설치하며, 약 15분 후 1단계가 완료되었다 

2. **클로드 메모리 관리 (골뱅이 기능 활용)** 
  1. 2단계 시작 전에 시스템 정비를 위해 클로드 마크다운 파일에 중요 파일들의 링크를 걸어준다 
  2. 파일 이름 앞에 **골뱅이(@)**를 붙여 링크를 걸면, 클로드가 이 파일들을 기억하게 된다 
  3. 클로드 마크다운을 읽다가 골뱅이 파일 경로가 나오면 그 파일도 읽어 들이는 원리이며, 이는 파일을 직접 읽으라고 지시하는 것보다 훨씬 정확하고 효과적이다 
  4. /memory를 입력해 보면 골뱅이로 넣어둔 파일들이 메모리에 잘 저장된 것을 확인할 수 있다 

3. **GitHub 연동 및 자동화 설정** 
  1. 활용도를 높이기 위해 Git으로 관리할 것이며, 지침 몇 가지를 추가한다 
  2. **GitHub 저장소 생성**: GitHub에 접속하여 새 저장소를 만들고 소스 공개 방지를 위해 **프라이빗**으로 설정한다 
  3. 생성된 저장소 주소를 복사하여 클로드 마크다운 파일에 저장소 주소라고 알려준다 
  4. **GitHub 액세스 토큰 생성**: 프로필 아이콘 > Settings > Developer Settings > Access Tokens 순서로 들어가 새 토큰을 생성하고 필요한 권한을 세팅한다 
  5. 생성된 토큰을 복사하여 지침 파일에 붙여 넣어 클로드에게 GitHub 접근 권한을 부여한다 
  6. 클로드에게 지금까지의 내용을 GitHub에 올려 달라고 요청했고, 6분 후 모두 업로드되었다 
  7. **자동 커밋**** 지침 추가**: "각 단계가 진행될 때마다 플랜 파일에 반영하고 GitHub에도 자동으로 올려 줘"라는 지침을 추가하여 편리성을 높인다 
  8. 이러한 방식은 다중 AI 간의 멀티 작업이 가능하게 하고, 컴퓨터 문제 발생 시 소스 전체를 복구할 수 있어 편의성과 안전성이 크게 향상된다 


## 5. 2단계 진행 (데이터베이스 설계 및 자동화) 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/496.jpg)
1. **2단계: ****데이터베이스**** 설계** 
  1. 2단계는 데이터베이스 설계이며, 플랜 모드에서 먼저 계획을 세우도록 한다 
  2. 데이터베이스 작업은 **Supabase MCP**가 알아서 처리하며, 실제로는 PostgreSQL 데이터베이스를 사용하지만 Supabase가 모든 것을 처리해 준다 
  3. 클로드는 데이터베이스 작업을 위해 상세 설계 문서인 LLD 파일을 참조한다 
  4. 계획이 생성되고 수락되자 클로드가 작업을 시작한다 

2. **Supabase를 통한 ****데이터베이스**** 자동 생성** 
  1. 클로드가 필요한 테이블들을 데이터베이스에 생성하고, 데이터베이스 속도를 빠르게 하기 위해 인덱스도 만들고 있다 
  2. 필요한 데이터들도 자동으로 삽입하며, 사용자가 손댈 필요 없이 전자동으로 진행된다 
  3. 이로써 데이터베이스 설계가 완료되었다 

3. **자동 업데이트 및 ****컨텍스트**** 저장** 
  1. 클로드는 지침에 따라 플랜 파일을 업데이트하고 GitHub에 자동 커밋하겠다고 보고한다 
  2. 사용자가 유료 플랜(20달러)으로 오래 작업하지 못하므로, 현재 컨텍스트 내용 전체를 파일로 옮겨 달라고 요청한다 
  3. 이때 **'****컨텍스트****'**라는 표현을 쓰는 것이 핵심이며, 현재 메모리에 있는 내용이 고스란히 파일로 저장된다 
  4. 클로드 실행 시 continue 옵션으로 대화를 이어갈 수도 있지만, 이 방법(파일 저장)이 더 선호된다 
  5. 파일이 만들어졌고, 약 130줄의 메모리 내용이 저장된 것을 확인할 수 있다 


## 6. 6단계 진행: 코덱스 CLI를 활용한 심층 분석 (투 트랙 전략) 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/579.jpg)
1. **6단계 진행 계획 및 코덱스 도입 배경** 
  1. 이 프로젝트는 총 일곱 단계로 구성되어 있으며, 계획 모드로 명령하여 벌써 5단계까지 진행되었다 
  2. 6단계는 오픈 AI의 API를 사용하는 부분이 있어, 이 부분에 **GPT-5**를 적용할 생각이다 
  3. 최신 API는 클로드가 가끔 실수할 수 있으므로, 이번에는 **코덱스(Codex) CLI**를 사용해 볼 것이다 
  4. 커서에서는 코덱스 CLI 익스텐션을 제공하며, 왼쪽 창에서 GPT-5 모델을 선택하고 명령하면 코덱스 CLI가 실행된다 

2. **코덱스 CLI****의 최적 환경 설정 (****WSL**** 활용)** 
  1. **문제점**: 윈도우에서 커서를 통해 코덱스 CLI를 실행하면 WSL이 아닌 윈도우 네이티브로 실행되는데, 윈도우 네이티브는 아직 불안정하여 성능이 많이 떨어진다 
  2. 윈도우 네이티브로 실행 시 한국어 오류 등 문제가 많았다 
  3. **해결책**: 코덱스는 WSL 환경에서 최고 성능을 발휘하므로, 클로드 코드처럼 터미널을 에디터 영역에 열고 wsl 입력 후 codex를 입력하여 코덱스 CLI를 띄운다 
  4. 이 방식으로 코덱스를 운영하면 성능이 좋으며, 한 탭에서 클로드 코드를 쓰다가 다른 탭으로 옮겨 코덱스 CLI를 동시에 사용할 수 있다 
  5. 커서 무료 플랜으로도 이 모든 기능이 제공되어 편의성이 극대화된다 

3. **코덱스 CLI**** 지침 설정 및 ****웹 조사** 
  1. 코덱스 CLI 실행 후, 그냥 명령어를 주는 대신 중요 파일들을 읽어 보라고 하여 프로젝트를 파악하게 돕는다 
  2. **에이전트 마크다운 파일**** 생성**: 코덱스용 지침 파일인 agent.md를 만들어 이 내용을 항상 기억하게 한다 
  3. 클로드에 골뱅이 기능이 없으므로, 다음 파일들을 읽어서 컨텍스트에 포함시키라고 직접 타이핑하여 주요 파일 네 개를 기억하게 한다 
  4. 지시 전에 /approver를 입력하고 **읽기 전용(Read Only) 모드**로 바꿔 계획을 세우기 좋게 한다 
  5. help_step_6 폴더를 만들고 6단계 진행을 위한 도움 파일(예제 코드 포함)을 만들도록 요청한다 
  6. **웹 조사**: GPT-5 최신 정보를 확인하기 위해 웹 조사도 함께 시켰으며, 코덱스가 클로드에 비해 상당히 깊이 있게 조사하는 모습을 보인다 
  7. 코덱스는 토큰 소모는 심하지만, 이런 심층 조사 능력 때문에 더 신뢰 있는 조사를 진행한다 
  8. 약 30분 후 작업이 완료되었고, help_step_6 폴더에 파일들이 생성되었다 

4. **클로드 코드****를 이용한 실제 코딩 (투 트랙 전략 실행)** 
  1. 사전 조사 결과물은 코덱스에게 맡겼으므로, 실제 코딩은 시원시원하게 작업하는 **클로드 코드**에게 맡긴다 
  2. 클로드 탭에서 사전 조사한 결과물을 참조하여 6단계 진행을 맡겼고, 드디어 6단계가 완료되었다 


## 7. 테스트, 오류 해결 및 최종 결과 
![image](https://resource-release.s3.ap-northeast-2.amazonaws.com/thumbnails/Ktm4XvPFubQ/763.jpg)
1. **테스트 및 오류 발생** 
  1. 실행 전 원활한 테스트를 위해 테스트용 자료를 넣어 달라고 요청했고, 테스트 데이터 생성이 완료되었다 
  2. 프로그램을 실행해 봤으나 역시나 에러가 많았고, 클로드 코드에게 하나씩 고쳐 달라고 요청했다 
  3. **해결되지 않는 문제**: 유독 회원 가입 기능(이메일 확인 링크)에서 계속 인증 오류가 발생했는데, 이는 코드 수정만으로 해결될 문제가 아니라 Supabase 서비스와 엮여 있어 복잡했다 

2. **Supabase 문제 해결을 위한 Context 7 MCP 활용** 
  1. 이 문제를 해결하는 데 **Context 7 MCP**가 큰 도움이 되었다 
  2. Context 7 MCP는 코딩 시 공식 설명서를 싹 긁어와서 정말 정확한 정보만 알려준다 
  3. 토큰 절약을 위해 **Playwright, Context 7, Supabase** 세 가지 MCP만 사용했으며, 이 조합은 레플릿 시스템에서 거의 필수템이다 
  4. 특히 Supabase 관련해서는 Context를 사용할 것을 강력히 추천한다 

3. **1차 완성된 가게부 앱 화면 및 성과** 
  1. 뚝딱뚝딱 작업하여 1차로 완성된 화면은 다음과 같다 
  1. **대시보드**: 전체 상황을 한눈에 볼 수 있다 
  2. **입력 페이지**: 직접 입력, 영수증, 문자 내역까지 넣을 수 있게 만들었다 
  3. **예산 관리**** 페이지**: 이번 달 예산을 얼마나 썼는지 바로 보인다 
  4. **리포트 페이지**: 월별 요약 정보를 도형과 도표로 깔끔하게 보여준다 
  2. 이 모든 것을 만드는 데 걸린 시간은 **딱 10시간**이었다 
  3. 입력 페이지 미완성 및 디자인 다듬기 등 부족한 점은 있지만, 10시간 만에 이 정도 성과는 만족도가 크다 

4. **Supabase의 역할과 AI 협업의 장점** 
  1. 10시간 만에 높은 성과를 낸 것은 Supabase가 큰 역할을 했기 때문이다 
  2. Supabase 로그인 후 프로젝트를 보면 컴퓨터에서 작업한 내용이 그대로 반영되어 있다 
  3. 테이블 메뉴에서는 데이터베이스 정보와 쌓인 데이터가 보이며, 편집/삭제 같은 관리도 바로 가능하다 
  4. Supabase는 데이터베이스 구조(데이터베이스 메뉴)와 가입 회원 목록(인증 메뉴)까지 보여준다 
  5. 이러한 기능들을 직접 만들었다면 양이 엄청났을 것이며, Supabase가 제공하는 기능을 가져다 쓰니 AI가 헛고생하지 않고 작업 속도와 품질이 향상된다 
  6. 이는 AI의 컨텍스트 크기 제한을 고려할 때 매우 좋은 시스템이다 
  7. 레플릿이 동일한 LLM을 쓰면서도 퀄리티가 좋은 이유는 미리 만들어진 코드 템플릿으로 최적의 환경을 세팅해 두었기 때문이며, 이 시스템을 구축하기 위해 많은 리서치를 했다 

5. **최종 AI 투 트랙 협업 시스템 추천** 
  1. 클로드 코드가 어려운 문제에 약해진 경향이 있으므로, **코드 분석은 무조건 ****코덱스 CLI**에게 시킨다 
  2. **실행은 ****클로드 코드**에게 맡기는 **투 트랙 전략**을 사용한다 
  1. 코덱스에게 폴더를 만들어 분석 결과 파일과 필요한 코드를 저장하게 한다 
  2. 클로드 코드에게 "이 폴더에 분석 파일 있으니까 이거 보고 처리해"라고 명령한다 
  3. 이로써 머리 쓰는 똑똑한 일(분석)은 GPT-5(코덱스)가 하고, 실행은 클로드 코드가 하는 환상의 짝꿍이 된다 
  4. 커서는 그 중간에서 다리 역할을 해 주어 작업이 훨씬 편해진다 
  5. 이 시스템을 꼭 한번 사용해 볼 것을 추천한다
